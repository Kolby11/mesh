## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| ChunkCache → disk | Stale cache entries must not survive a .mesh source change; eviction is best-effort (FNV64 collisions would keep stale entry but are astronomically unlikely on source text) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-95-03 | Tampering | ChunkCache stale entry | mitigate | `reload_source()` evicts by FNV64 content hash before recompile; next `compile_and_execute` does fresh `get_or_insert` from disk. FNV64 collision rate on source text is ~1 in 2^64 — acceptable. |
| T-95-04 | Denial of Service | ChunkCache unbounded growth | accept | Cache is in-memory only, bounded by number of unique source strings. No eviction on LRU — accepted for v1.17 scope. |
</threat_model>
