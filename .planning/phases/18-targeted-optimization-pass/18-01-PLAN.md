# Plan 18-01: Fresh Baseline and Hotspot Ranking

<objective>
Capture a fresh Phase 18 baseline from the existing debug profiling and benchmark contract, rank canonical benchmark/stage costs by absolute latency, and select the one hotspot that Phase 18 will optimize.
</objective>

<requirements>
- `OPT-01`: At least one high-impact hotspot identified by the new profiler is optimized within `v1.3`.
</requirements>

<decision_coverage>
- `D-01`: fresh profiler measurements are captured at the start of the phase.
- `D-02`: the selected target is the worst measured canonical benchmark or stage.
- `D-03`: largest absolute latency is the tie-breaker.
- `D-04`: backend paths require visible frontend impact.
</decision_coverage>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Baseline is stale or speculative | high | Write `18-BASELINE.md` during this plan with fresh values and commands used. |
| Target selection is biased toward easy work | medium | Include a ranked table by absolute latency and select the top eligible hotspot. |
| Backend-only timing is selected without visible impact | medium | Baseline artifact must include frontend surface timing for backend candidates. |
</threat_model>
