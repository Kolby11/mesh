# Phase 102 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Phase nature:** Backend infrastructure — no traditional UI components built. This contract captures the visual rendering quality guarantees and system-level "interaction" behaviors (monitor hotplug, compositor protocol handling). The design system is MESH's existing Material 3 token-based theme rendered through Skia.

## Spacing Scale

Declared values (inherited from existing MESH theme system):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

**Phase 102 contract:** Spacing tokens and layout calculations remain in **logical CSS pixels** throughout the entire pipeline. The scale factor is only applied at the buffer allocation boundary — `PixelBuffer::new(physical_width, physical_height)` where `physical = ceil(logical × scale)`. This means a `md` token of 16px translates to 32 device pixels at 2× scale, but the layout engine computes with `16.0`.

**Source:** CONTEXT.md "layout coordinates stay in logical CSS pixels throughout"

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | Regular (400) | 1.5 |
| Label | 12px | Medium (500) | 1.4 |
| Heading | 20px | Semibold (600) | 1.2 |
| Display | 28px | Bold (700) | 1.1 |

**Phase 102 contract:** Text is rasterized at physical pixel density via Skia/Parley. At 2× integer scale, glyphs are drawn at native resolution — no bitmap upscaling occurs. At 1.5× fractional scale, the Skia surface renders at `ceil() × 1.5` physical pixels and the compositor's `wp_viewporter` downscales to logical dimensions.

**Explicit guarantee:** Text and icons SHALL appear sharp without upscaling artifacts on integer-scale HiDPI displays (success criterion 1).

**Deferred:** Font rendering subsystem work (higher-DPI font metrics, hinting adjustments) — separate concern per CONTEXT.md.

**Source:** ROADMAP.md success criterion 1; CONTEXT.md deferred section.

## Copywriting Contract

| Error state | N/A |

**Phase 102 contract:** This phase produces zero user-visible text, CTAs, or interactive prompts. It is a rendering infrastructure phase. The only "interaction" is system-level:

- **Scale change (hotplug):** Compositor sends `wp_fractional_scale_v1::preferred_scale` or `wl_surface::preferred_buffer_scale` event → MESH resizes surface and triggers full redraw. No user prompt or confirmation. Behavior is invisible to the end user — they see the shell surface resize smoothly.
- **Fallback (no fractional-scale protocol):** MESH uses `wl_output::scale` integer fallback. No user notification. Rendering remains correct, just at integer scale.
- **Fallback (no viewporter + fractional scale):** MESH rounds to nearest integer and uses `set_buffer_scale()`. Slight sizing mismatch accepted. No user notification.

## Visual Quality Contracts

These are the effective "UI" guarantees of this phase — the visual correctness properties that must hold:

### VC-01: Integer Scale Sharpness
At any integer scale factor (1×, 2×, 3×), glyphs, icons, and vector geometry SHALL render at native physical pixel resolution without visible upscaling artifacts. The `PixelBuffer` is allocated at `logical × scale` physical pixels.

### VC-02: Fractional Scale Correctness
At non-integer scale factors (e.g., 1.25×, 1.5×, 1.75×), the `PixelBuffer` is allocated at `ceil(logical × scale)` physical pixels. `wp_viewporter::set_destination()` sets the compositor viewport to logical dimensions so the compositor scales the oversampled buffer down. The result SHALL be visually sharper than integer-rounding.

### VC-03: Hotplug Continuity
When a monitor is connected or disconnected (scale factor change), the surface SHALL resize and redraw without stale pixels, visual tearing, or transient corruption. Any frame rendered after the scale change SHALL use the new scale factor for buffer allocation and present coordinates.

### VC-04: Damage Rect Scaling
Damage rects from Phase 101 are authored in logical CSS pixel coordinates. Before `wl_surface::damage_buffer` calls, rects SHALL be scaled to physical coordinates: `rect.x = (logical_x * scale) as u32`, same for y, width, and height. This ensures the compositor's per-rect damage tracking is accurate at all scale factors.

### VC-05: Layout Immutability
Layout engine output, CSS coordinates in the display list, and theme token spacing values SHALL NOT change when scale factor changes. Only the final buffer dimensions and present-time coordinate scaling adjust.

### VC-06: Fallback Fidelity
When `wp_fractional_scale_v1` is unavailable, the integer `wl_output::scale` path SHALL produce a correctly-scaled buffer sized at `logical × integer_scale` — no rendering corruption, no protocol errors, no visual regressions compared to pre-Phase 102 behavior.

## System Interaction Model

| Event | Source | MESH Behavior | User-Visible Effect |
|-------|--------|---------------|---------------------|
| `preferred_scale(180)` | `wp_fractional_scale_v1` | Set `scale = 1.5`, allocate buffer at `ceil(logical × 1.5)`, set viewport destination, full redraw | Shell surface immediately renders at 1.5× native density |
| `preferred_buffer_scale(2)` | `wl_surface` compositor event | Set `scale = 2.0`, allocate buffer at `logical × 2`, set_buffer_scale(2), full redraw | Shell surface snaps to 2× integer scale |
| Monitor hotplug (new scale) | Output state change | Detect scale delta, trigger surface resize + full redraw with new scale | Surface resizes and sharpens to new density |
| No `wp_fractional_scale_v1` | Protocol negotiation failure | Use `wl_output::scale` integer, `set_buffer_scale()` only | Shell renders at integer scale — correct but may appear slightly larger/smaller than fractional |
| No `wp_viewporter` + fractional scale | Protocol negotiation failure | Round to nearest integer, use `set_buffer_scale()` only | Shell renders at nearest integer scale — slight sizing mismatch accepted |
