# Plan 33-01: Locale-Aware Access-Key Resolution

<objective>
Extend Phase 32 keybind declarations with typed per-locale trigger defaults and resolve effective keybinds from user overrides, exact locale, parent locale, and generic defaults while preserving current shortcut behavior.
</objective>

<requirements>
- `LOCL-01`: The shell resolves keybinds from user overrides, locale-specific defaults, and generic module defaults in deterministic precedence order.
- `LOCL-02`: Modules can define localized access keys, including English `Accept -> A` and Slovak `Prijat -> P` style mappings.
- `LOCL-03`: Missing locale-specific bindings fall back to generic defaults without breaking the action.
</requirements>

<must_haves>
- Manifest `KeybindAction` exposes `localized_triggers` as a typed map keyed by locale id.
- Localized entries override trigger data only and do not alter handler, target, scope, label, or label_i18n_key.
- Resolver precedence is user override, exact locale, parent locale, generic trigger, then no binding.
- Parent fallback supports regional locales such as `sk-SK -> sk`.
- Phase 33 localizes access-key triggers only; shortcut actions keep generic shortcut defaults unless a user override is present.
- Blank or incomplete localized entries fall back silently; diagnostics remain out of scope for Phase 33.
- Existing navigation-bar shortcut behavior and stable action-id override behavior remain compatible.
</must_haves>

<truths>
- Stable action id is the override identity; translated labels must not participate in override lookup.
- `localized_triggers` is a per-action field, not a top-level locale table.
- Phase 33 must not implement conflict diagnostics, richer script dispatch payloads, accessibility metadata, global shortcuts, or a settings UI.
</truths>

<threat_model>
No new external input source is added, but module-authored manifests can contain malformed localized triggers. Phase 33 must treat localized trigger data as untrusted: parse into typed structures, ignore blank/incomplete localized entries during resolution, preserve user overrides as the highest precedence, and keep generic defaults available. The main regression risks are changing existing shortcut behavior, using translated labels as identity, or capturing shortcut keys that should stay generic; tests must prove those do not happen.
</threat_model>

<action>
Update shell-side keybind resolution so typed manifest actions can choose effective triggers by user override, active locale, parent locale, and generic default.

In `crates/core/shell/src/shell/component/input/keyboard.rs`:
- Extend internal declaration data so manifest declarations keep:
  - `action_id: String`
  - `generic_trigger: mesh_core_module::KeybindTrigger`
  - `localized_triggers: HashMap<String, mesh_core_module::KeybindTrigger>`
  - `handler: String`
  - `target_ref: Option<String>`
- Legacy settings declarations may keep generic behavior and should have an empty localized trigger map.
- Add a resolution source enum for internal test assertions:
  - `UserOverride`
  - `LocaleDefault { locale: String }`
  - `ModuleDefault`
- Extend `ResolvedSurfaceShortcut` with:
  - `trigger_kind: mesh_core_module::KeybindTriggerKind`
  - `source: KeybindResolutionSource`
- Add helper `fn keybind_locale_candidates(active_locale: &str) -> Vec<String>`.
  - `keybind_locale_candidates("sk-SK")` returns `vec!["sk-SK".to_string(), "sk".to_string()]`
  - `keybind_locale_candidates("sk")` returns `vec!["sk".to_string()]`
  - blank input returns an empty vector
- Add helper `fn localized_access_key_trigger(action_trigger_kind, localized_triggers, active_locale) -> Option<(KeybindTrigger, String)>` or equivalent.
- Effective trigger rules:
  - if `KeyboardSettings.surface_shortcuts[surface_id][action_id].key` is `Some(nonblank)`, return that key with `source = UserOverride`
  - if generic trigger kind is `AccessKey`, inspect `localized_triggers` for exact active locale then parent locale
  - localized trigger candidate is usable only when `kind == AccessKey` and `key` is present and nonblank
  - if generic trigger kind is `Shortcut`, ignore `localized_triggers`
  - fallback to generic trigger when generic `key` is present and nonblank with `source = ModuleDefault`
  - otherwise omit the binding
- Preserve existing dispatch order in `crates/core/shell/src/shell/component/input/mod.rs`; do not move `dispatch_surface_shortcut`.

In `crates/core/shell/src/shell/component/tests/interaction/navigation.rs`:
- Add `fn keybind_locale_exact_locale_wins_over_parent_and_generic`.
- Add `fn keybind_locale_parent_locale_wins_over_generic`.
- Add `fn keybind_locale_user_override_wins_over_locale_and_generic`.
- Add `fn keybind_locale_blank_localized_trigger_falls_back_to_generic`.
- Add `fn keybind_locale_shortcut_ignores_localized_trigger`.
- Tests should construct `KeybindAction` records with `trigger.kind = AccessKey` for localized access-key cases and `trigger.kind = Shortcut` for the shortcut-ignore case.
- Tests should assert `ResolvedSurfaceShortcut.key`, `trigger_kind`, `action_id`, and `source`.
</action>
