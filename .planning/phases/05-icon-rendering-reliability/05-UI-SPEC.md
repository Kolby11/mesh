# Phase 05 - UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none - MESH `.mesh` surfaces use project theme tokens and built-in primitives, not shadcn |
| Preset | not applicable |
| Component library | none - built-in `box`, `row`, `column`, `button`, `slider`, `text`, and `icon` elements |
| Icon library | MESH semantic icon registry backed by configured XDG icon packs and a core-owned built-in vector fallback |
| Font | `token(typography.family)` = Inter in default dark/light themes |

Source: shadcn/design-system absence verified by repo scout; theme tokens verified in `config/themes/mesh-default-dark.json`, `config/themes/mesh-default-light.json`, and existing core surface styles.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline control padding, nav-rail label gaps |
| sm | 8px | Compact element spacing, panel padding, chip/button padding |
| md | 16px | Default element spacing, drawer padding, panel section gaps |
| lg | 24px | Quick settings surface padding, navigation shell horizontal padding |
| xl | 32px | Wider layout gaps when needed by future surface expansion |
| 2xl | 48px | Major section breaks; maps to existing `spacing.3xl` |
| 3xl | 64px | Page/surface-level spacing; maps to existing `spacing.4xl` |

Exceptions: Existing dense metadata stack micro gaps are not part of the Phase 05 spacing contract because this phase does not modify those implementation details. Existing icon lookup hints of 16px and 18px may remain because `size` is a lookup hint, not the painted box contract. Existing painted icon boxes may remain at their current control-specific dimensions: panel 16x16, quick settings 18x18, battery 24x24, nav settings/theme 28x26, nav volume 30x28. Interactive icon buttons must preserve their current stable boxes: 34x32, 36x36, 44x40, or 44px-wide variants depending on the existing surface.

Source: default theme spacing tokens and current surface CSS. Defaults only apply to any new spacing introduced in this phase.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `token(color.surface)` / dark `#1C1B1F` / light `#FFFBFE` | Panel, quick settings, navigation shell backgrounds |
| Secondary (30%) | `token(color.surface-container)` / dark `#211F26` / light `#F3EDF7` | Drawer, nav controls, button backgrounds, compact status containers |
| Accent (10%) | `token(color.primary-container)` / dark `#4F378B` / light `#EADDFF` | Active quick-settings nav item, hover-promoted icon buttons, active/selected control states, slider emphasis |
| Destructive | `token(color.error)` / dark `#F2B8B5` / light `#B3261E` | Destructive actions only; no destructive actions are introduced by this phase |

Accent reserved for: active quick-settings navigation, hover-promoted icon controls, selected/active controls, and slider emphasis. Missing-icon fallback must not use accent; it must inherit the surrounding text/icon color and render as a neutral degraded visual.

Visual hierarchy: Primary visual anchor remains the existing active control state; icon fallback must stay visually subordinate and never become the focal point.

Source: default dark/light theme tokens and existing core surface styles.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Reload icons |
| Empty state heading | Icon unavailable |
| Empty state body | The configured icon profile did not provide this semantic icon. MESH rendered the built-in fallback and kept the surface active. |
| Error state | Icon profile failed to load. Check the icon config paths and keep the previous active profile or built-in fallback active. |
| Destructive confirmation | none - Phase 05 introduces no destructive user action |

Copy scope: there is no new visible settings UI in this phase. The copy above is reserved for diagnostics, degraded health summaries, CLI/config reload surfaces if they exist, and test-visible fallback assertions. Existing panel, quick settings, and navigation labels remain unchanged.

Source: Phase 05 context decisions D-08 through D-12 and the phase boundary that config-level profile switching is enough.

## Phase-Specific Icon Rendering Contract

| Contract Area | Required Behavior | Source |
|---------------|-------------------|--------|
| Icon naming | Core surfaces must author semantic MESH names only, for example `audio-volume-muted`, `network-wireless`, `settings`, `battery-empty`; pack-specific names and filesystem paths stay in icon config/mapping. | CONTEXT D-01, D-21 |
| Mapping | A dedicated icon config owns active profile, configured pack roots, pack IDs, and ordered fallback lists. First available candidate wins. | CONTEXT D-02 through D-06 |
| Painted size | CSS/layout box width and height control painted size. The `size` attribute is only a resolver lookup hint. | CONTEXT D-15; `render_icon_node()` |
| Default color | Monochrome SVG and raster icons inherit `style.color` from their surrounding surface, same as text. | CONTEXT D-13, D-17 |
| Multicolor opt-out | Assets marked multicolor preserve source colors and ignore inherited tint. Use only for logos, flags, and similar assets. | CONTEXT D-14 |
| SVG fidelity | Support common symbolic SVGs by rasterizing through the render pipeline and tinting by alpha mask. Full complex SVG fidelity is out of scope. | CONTEXT D-16 |
| Raster fidelity | Decode PNG/JPG/JPEG/BMP, resize to the layout box, and apply alpha-mask tint unless multicolor is true. | REQUIREMENTS ICON-03; RESEARCH |
| Missing fallback | Missing icons render a neutral built-in vector fallback inside the same icon box, with transparent background and no layout shift. | CONTEXT D-09, D-11 |
| Diagnostics | Missing icon diagnostics dedupe by plugin ID plus semantic icon name and mark plugin health warning/degraded without unloading the plugin. | CONTEXT D-10, D-12 |
| Profile switching | Config-level profile switching is sufficient. It must invalidate icon resolution caches so surfaces repaint the selected profile without `.mesh` call-site changes. | CONTEXT D-20; RESEARCH |

## Interaction Contract

Icon rendering must be passive: hover, active, click, slider, and service update interactions remain owned by the existing surface components. This phase must not introduce icon-specific click targets, visible profile switchers, or special-case event handlers.

State changes that alter semantic icon names, such as volume percent, mute state, theme mode, or battery level, must repaint the new icon without resizing the surrounding control. Missing icon fallback must behave exactly like a painted icon for hit testing: the parent button remains clickable and accessible labels/titles remain the existing text.
