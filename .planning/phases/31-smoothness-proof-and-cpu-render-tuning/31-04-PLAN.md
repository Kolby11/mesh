# Plan 31-04 - Close Retest Audio Trigger and Mute Consistency Gaps

## Objective

Close the two remaining Phase 31 live UAT failures after 31-03: same-hover volume-trigger close and persistent mute state mismatch between the audio popover and navigation bar.

## Gap Sources

- `.planning/phases/31-smoothness-proof-and-cpu-render-tuning/31-UAT.md` tests 2 and 5
- `.planning/debug/phase31-live-uat-diagnosis.md`

## Diagnosis Summary

31-03 fixed pointer-open focus theft and the immediate slider grab path. The remaining failures are narrower:

- Test 2 now works after pointer hover leaves and re-enters the volume button, which points to the close branch relying on portal hidden-state/tick bookkeeping instead of issuing an immediate shell hide request for the open popover.
- Test 5 still has duplicate mute reconciliation: the shell owns canonical optimistic `mesh.audio.muted`, but the popover also owns local `pending_muted_state`/`audio_muted` while the navigation button renders service state directly.
