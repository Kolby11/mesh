# Plan 31-03 - Close Remaining Live Audio Interaction Gaps

## Objective

Close the remaining Phase 31 live UAT failures after 31-02 by fixing first-input behavior around audio popover activation and unifying mute pending-state reconciliation across the navigation button and audio popover.

## Gap Sources

- `.planning/phases/31-smoothness-proof-and-cpu-render-tuning/31-UAT.md` tests 2, 3, and 5
- `.planning/debug/phase31-live-uat-diagnosis.md`

## Diagnosis Summary

31-02 fixed stale slider repainting and added lower-level coverage, but live UAT still fails at the shell interaction boundary:

- Opening the audio popover leaves the next navigation-bar click unreliable because focus/keyboard ownership cleanup is still coupled to the first pointer press after popover activation.
- The slider no longer lags, but the first drag immediately after opening does not bind; the existing first-grab test starts from an already painted component and misses the show/focus/first-input sequence.
- The navigation volume button still renders backend `audio.muted` directly while the popover uses local pending mute state and non-idempotent `toggle_mute`, so stale backend updates can make the second toggle appear inverted.
