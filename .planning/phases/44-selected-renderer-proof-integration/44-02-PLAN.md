# Plan 44-02 - Shell Proof Observability Integration

<objective>
Wire the focused proof snapshot into the existing component paint path so retained identity, dirty categories, damage/profiling payloads, and non-fatal diagnostics remain visible through current shell/component boundaries.
</objective>

<must_haves>
- D-02: Keep existing renderer and presentation ownership.
- D-04: Preserve MESH stable node IDs.
- D-05: Preserve typed invalidation categories.
- D-06: Preserve existing damage/profiling observability.
- D-07: Record non-fatal diagnostics for adapter limitations.
</must_haves>

<threat_model>
- T-44-02-01 Proof integration erases present damage: High. Mitigate by building the proof snapshot without changing `paint_damage`, `selected_paint`, `last_present_damage`, or `ProfilingInvalidationSnapshot` behavior.
- T-44-02-02 Diagnostics become fatal: High. Mitigate by recording proof diagnostics through `Diagnostics` and never returning `ComponentError` only because the proof has diagnostics.
- T-44-02-03 Test-only API leaks into product UX: Medium. Mitigate by keeping the proof snapshot component-local and exposing it only through `#[cfg(test)]` helper methods unless an existing debug path already needs it.
</threat_model>
