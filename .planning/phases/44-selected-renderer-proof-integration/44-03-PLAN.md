# Plan 44-03 - Text Selection and AccessKit Boundary Proof

<objective>
Prove text layout, selection geometry/colors, and AccessKit-compatible retained-node updates through the focused proof path.
</objective>

<must_haves>
- D-08: Preserve theme-owned `color.selection-background` and `color.selection-foreground`.
- D-09: Test selection geometry and paint behavior through the selected path.
- D-10: Derive AccessKit-compatible updates from MESH retained node IDs.
</must_haves>

<threat_model>
- T-44-03-01 Selection color authority moves into proof code: High. Mitigate by asserting `_mesh_selection_background` and `_mesh_selection_foreground` remain the source of painted selection colors.
- T-44-03-02 Accessibility boundary uses unstable generated order: High. Mitigate by deriving every accesskit ID from `NodeId`, not traversal index.
- T-44-03-03 Text proof becomes label-only: Medium. Mitigate by asserting selection geometry and paint tests still execute with focused proof records present.
</threat_model>
