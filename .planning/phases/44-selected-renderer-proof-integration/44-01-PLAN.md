# Plan 44-01 - Focused Proof Snapshot Adapter

<objective>
Add the render-crate proof snapshot boundary that converts current MESH retained data into focused layout, text, paint, and accessibility evidence while keeping `mesh_core_elements::NodeId` authoritative.
</objective>

<must_haves>
- D-01: Advance only the MESH-owned focused-crate path.
- D-02: Keep the proof behind current renderer ownership.
- D-04: Preserve MESH stable node IDs as the identity source.
- D-05: Preserve typed invalidation categories.
- D-10: Expose an AccessKit-compatible retained-node update boundary.
- D-11: Keep focused crate adoption narrow and avoid broad dependency expansion.
- D-12: Keep the adapter local and reversible.
</must_haves>

<threat_model>
- T-44-01-01 Identity drift: High. Mitigate by storing every proof node and paint/accessibility record with the original `NodeId` plus `stable_node_id = node_id.to_string()`.
- T-44-01-02 Untyped proof output: High. Mitigate by copying render-object dirty counters into explicit `FocusedDirtyEvidence` fields for geometry, material, text, and accessibility.
- T-44-01-03 Premature crate lock-in: Medium. Mitigate by naming evidence fields `taffy_layout`, `parley_text`, and `accesskit_node_id` without requiring broad renderer replacement.
</threat_model>

      - Add a `FocusedProofDiagnostic` with message `focused proof node has zero-size layout` for each node whose layout width or height is `0.0`.
