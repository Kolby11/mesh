<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| compositor→mesh | Blur manager global availability is untrusted — gracefully handles `None` |
| mesh→compositor | Blur region coordinates come from display list (trusted source) |
| mesh→compositor | Region rect values passed to `wl_region::add` use logical coordinates per protocol spec |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-103-01 | Tampering | `attach_shm_buffer → kde_blur.set_region` | mitigate | Region rect clamped to positive i32 before `wl_region::add` — out-of-bounds coordinates would be a protocol error |
| T-103-02 | Denial of Service | `configure() → manager.get_blur()` | mitigate | `KdeBlur` object created once per surface (lazily), not per frame — unbounded object creation is not possible |
| T-103-03 | Elevation of Privilege | `org_kde_kwin_blur_manager` global bind | accept | Protocol is informational (blur hint); untrusted compositor cannot escalate — it can only choose not to blur |
</threat_model>
