<objective>
Remove CPU-side software blur rendering per BLUR-03. The `apply_backdrop_filter` methods become no-ops and `push_backdrop_filter_command` becomes a no-op. Call sites are preserved so future `backdrop-filter` effects can be re-wired at the same insertion points. The `VisualFilter` data continues to flow through the display list unchanged so Plan 02's compositor blur region computation works correctly.

Purpose: Eliminate the CPU blur rendering path. On KDE, the compositor handles blur (Plan 01-02). On non-KDE, surfaces render with a flat background — no error, no CPU blur fallback, because clients cannot read the compositor framebuffer.
Output: Modified `painter.rs` and `tree.rs` with no-op blur rendering.
</objective>
