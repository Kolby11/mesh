<objective>
Replace `RetainedNodeSnapshot.state_hash: u64` with `state: ElementState` so the retained tree stores the actual interaction state per-node instead of just a hash. Augment `diff_flags()` to compute which specific state bits flipped (`changed_state_bits`) between frames. Add `changed_state_bits: u32` to `RetainedTreeDirtySummary` to surface which pseudo-classes changed in the summary, enabling the targeted restyle logic in Plan 3.

Purpose: Per D-02, the retained tree currently only knows that "state changed" via hash comparison. To enable targeted restyle, we need to know *which* state bits changed (`:hover`, `:focus`, `:active`). Storing `ElementState` directly and diffing old/new enables computing `changed_state_bits = old_state_mask ^ new_state_mask` — a bitmask of flipped pseudo-classes.

Output: `RetainedNodeSnapshot` stores `state: ElementState` instead of `state_hash: u64`. `diff_flags()` returns `(RetainedNodeDirtyFlags, u32)` where the u32 is the changed_state_bits mask. `RetainedTreeDirtySummary` gains a `changed_state_bits: u32` field accumulated during `update()`.
</objective>

```rust
/// Converts ElementState to a u32 bitmask using the same bit positions
/// as the style resolver's STATE_HOVERED, STATE_FOCUSED, etc. constants.
fn state_bitmask(state: ElementState) -> u32 {
    let mut mask = 0u32;
    if state.hovered      { mask |= 1 << 0; }
    if state.focused       { mask |= 1 << 1; }
    if state.active        { mask |= 1 << 2; }
    if state.disabled      { mask |= 1 << 3; }
    if state.read_only     { mask |= 1 << 4; }
    if state.required      { mask |= 1 << 5; }
    if state.selected      { mask |= 1 << 6; }
    if state.checked       { mask |= 1 << 7; }
    if state.expanded      { mask |= 1 << 8; }
    if state.pressed       { mask |= 1 << 9; }
    if state.invalid       { mask |= 1 << 10; }
    if state.value         { mask |= 1 << 11; }
    if state.focus_visible { mask |= 1 << 12; }
    mask
}
```

This mirrors the `active_state_mask` function in resolve.rs (line 1052) but is self-contained in runtime_tree.rs to avoid a cross-crate dependency on private constants.
