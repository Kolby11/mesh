<objective>
Add field-level service-event fan-out filtering to `handle_service_event()`
in `shell_component.rs`. Before overwriting `cached_service_payloads` with
the new payload, save the previous payload, then run a JSON key-level diff
to extract changed `(service, field)` pairs. For each pair, query
`self.node_service_field_deps.nodes_reading_field(service, field)` (the
bidirectional index from Phase 97). If every query returns an empty set,
skip invalidation entirely — the component has no template node that reads
any changed field. Otherwise, call `invalidate_script_state_narrow()` (added
in Plan 01) instead of the unconditional `invalidate_script_state()` so the
narrow path can take effect. The existing Lua-side
`tracked_service_fields_changed()` check is preserved as a fallback path.

Purpose: Satisfies INV-02 — service events fan out only to components whose
tracked field sets intersect the changed fields.

Output: Modified `shell_component.rs`, new `tests/invalidation/service_fanout.rs`,
updated `tests/invalidation.rs` mod registration.
</objective>
