  <behavior>
    - Implement a Skia layer stack for `PainterCommand::PushLayer` and `PainterCommand::PopLayer`.
    - Support `PainterLayer { opacity, blend_mode: PainterBlendMode::SrcOver, filter: PainterFilter::None }`.
    - Support `PainterLayer` with `PainterFilter::Blur(VisualFilter { blur_radius })` by using a Skia image filter on the save layer.
    - Keep unsupported non-`SrcOver` blend modes diagnostic until explicitly executed.
    - Change Skia capabilities to `layers: true` only after tests pass.
  </behavior>
  <action>
    In `SkiaPaintBackend::execute_commands`, maintain a stack of Skia save counts for layer commands. For `PushLayer`, compute the effective clip from the command bounds and clip stack, call `canvas.save_layer` with a `SaveLayerRec` whose bounds match that effective rectangle, whose alpha is derived from `PainterLayer.opacity`, and whose image filter is Skia blur when `PainterFilter::Blur` is non-none. For `PopLayer`, restore the last saved layer count. Add tests named `skia_effect_layer_opacity_isolates_child_pixels` and `skia_effect_layer_blur_expands_painted_pixels`.
  </action>
  <verify>
    cargo test -p mesh-core-render skia_effect_layer -- --nocapture

    - `rg "layers: true" crates/core/frontend/render/src/surface/painter/backend.rs` finds updated capability reporting.

  </acceptance_criteria>

  <behavior>
    - Implement `DrawLinearGradient` with a Skia linear gradient shader from top to bottom.
    - Implement `DrawImage` for `PainterImageSource::Path` using existing image loading/raster cache patterns where practical.
    - Respect command clips for both image and gradient drawing.
    - Change Skia capabilities to `images: true` only after image tests pass.
  </behavior>
  <action>
    Add Skia execution helpers `draw_linear_gradient_command` and `draw_image_command` in `backend.rs`. The gradient helper should draw a rect or rounded rect bounded by the command `rect` and `radius`, clipped by the command `clip`, using `gradient.from` at the top edge and `gradient.to` at the bottom edge. The image helper should load the relative path from `PainterImageSource::Path`, decode it into an RGBA image, scale it into the command rect, and draw it clipped. If existing private image cache helpers in `icon.rs` cannot be reused directly, add a small crate-private helper there or keep a local backend helper with freshness-safe loading. Add tests named `skia_effect_linear_gradient_draws_top_and_bottom_colors`, `skia_effect_image_draws_source_pixels`, and `skia_effect_image_respects_command_clip`.
