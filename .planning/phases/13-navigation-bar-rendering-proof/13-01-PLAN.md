# Plan 13-01 - Shell Surface Restructure and Selectable Status Cluster

## Objective

Turn the top-level navigation bar from a control-only strip into a shell-like proof surface with one compact passive status cluster plus one control cluster, and put the opt-in selectable text proof directly on the primary surface.

## Design Truths

- The navigation bar remains shell chrome, not a dashboard or a new feature surface.
- Passive status copy is the approved place for selection proof; interactive control text remains non-selectable.
- Secondary copy may collapse first under constrained width, but the bar should still read as status plus controls.

<threat_model>
T-13-01: The migration could satisfy "richer bar" by stuffing interactive-looking text into the main surface, which would blur selection boundaries and make control hit targets ambiguous.
Mitigation: keep the primary proof text in an explicit passive status cluster, mark only that text `selectable="true"`, and keep control affordances isolated in the control cluster.
</threat_model>
