### 1b: Update return_slot to call Thread::reset()

with:
```rust
    pub(crate) fn return_slot(&mut self, slot_index: usize, lua: Lua) {
        // ISO-03: Reset thread state to prevent state bleed to the next component
        // that checks out this slot. Clears the Lua stack, any pending yields,
        // and resets the thread status.
        lua.current_thread()
            .reset()
            .expect("Thread::reset failed during pool slot return");
        self.slots[slot_index] = Some(lua);
    }
```

The `lua` parameter is owned (`Lua`), so we can call `current_thread()` which returns a `Thread` borrow. The `expect()` is appropriate because `Thread::reset` should not fail on a sandboxed, idle thread — if it does, it indicates internal Lua state corruption and a panic is the correct response.

### 1c: Verify PooledVm::Drop still works
The `PooledVm::Drop` impl calls `POOL.with(|cell| cell.borrow_mut().return_slot(idx, lua))`. Since `return_slot` now calls `Thread::reset()` which borrows from the `Lua`, and we own the `Lua` at that point, this is fine — the reset happens before the Lua is moved into the slot.

    }

| Boundary | Description |
|----------|-------------|
| return_slot → Thread::reset() | Thread state (stack, yield status) is cleared before slot reuse |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-94-05 | Information Disclosure | Thread::reset() in return_slot | mitigate | `Thread::reset()` clears the Lua stack and any pending state from the previous script execution. A subsequent checkout gets a clean thread. The `expect()` panics on failure — internal Lua state corruption is not recoverable. |
| T-94-06 | Denial of Service | ChunkCache unbounded growth | accept | ChunkCache grows with every unique source hash. In practice, the number of unique .mesh files in a MESH installation is bounded (tens to low hundreds). No eviction policy needed for Phase 94; Phase 95 adds hot-reload eviction. |
</threat_model>
