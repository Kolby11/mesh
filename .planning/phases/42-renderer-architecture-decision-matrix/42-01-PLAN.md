# Plan 42-01 - Source Inventory and Scorecard Frame

<objective>
Create the source inventory and empty decision matrix frame that Phase 42 will use to compare Blitz direct adoption, Blitz-inspired architecture borrowing, and the MESH-owned focused-crate renderer path. Covers REND-01 and REND-02 by making every required scorecard dimension and hard blocker explicit before outcomes are filled.
</objective>

<must_haves>
- D-01: Treat Blitz as the reference architecture by default while preserving MESH-owned renderer boundaries unless direct adoption is clearly cleaner.
- D-02: Direct adoption may justify internal MESH redesign only when the matrix makes the redesign explicit and measurable.
- D-03: Wayland shell model mismatch is a hard blocker for direct Blitz adoption.
- D-04: Browser-engine-level performance overhead is a hard blocker for direct Blitz adoption.
- D-06: Direct Blitz adoption has two hard blockers: Wayland shell model fit and no browser-engine-level performance overhead.
- D-07: After blockers pass, capability gain matters most.
- D-08: Performance overhead includes interaction latency, render/layout/paint architecture cost, startup time, compile/build cost, binary size, memory, resource usage, and native dependency burden.
- D-09: Observability may temporarily regress during prototype work only if final migration restores MESH equivalents for invalidation, damage, profiling, diagnostics, and debug payloads.
- D-10: Build/dependency cost can be accepted only when significant renderer capability is unlocked and the cost is measured.
</must_haves>

<threat_model>
- T-42-01 Source drift or unsourced claims: Medium. Mitigate by listing one primary source URL per external candidate in `42-SOURCE-INVENTORY.md`.
- T-42-02 Hidden browser-engine adoption: High. Mitigate by separating hard blockers from weighted scorecard dimensions in `42-DECISION-MATRIX.md`.
- T-42-03 Loss of existing observability contracts: Medium. Mitigate by making retained invalidation, profiling, diagnostics, and damage explicit scorecard dimensions.
</threat_model>
