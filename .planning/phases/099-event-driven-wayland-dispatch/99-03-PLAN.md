<objective>
Integrate event-driven blocking dispatch into the shell loop. Replace `std::thread::sleep(sleep_for)` at the bottom of `Shell::run()` with a call to `PresentationEngine::wait_for_events()` when running the Wayland backend, while preserving the `std::thread::sleep` path for the dev-window backend. Create the eventfd that IPC/backend paths will signal (Plan 04 adds the signaling). Remove the `MAX_IDLE_SLEEP` clamp so the scheduler can block for the full computed deadline. Record `SchedulerIdle` profiling stage after each block.

Purpose: Eliminate idle CPU burn while preserving existing loop ordering (drain → tick → render → present → block) and dev-window compatibility.
Output: Shell loop that blocks on real Wayland events instead of burning CPU, with profiling visibility.
</objective>
