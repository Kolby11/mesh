# Phase 04 - UI Design Contract

> Visual and interaction contract for Phase 4 real core surfaces. Generated by gsd-ui-phase, verified inline against the six UI-SPEC dimensions.

---

## Scope

Phase 4 touches two shipped shell surfaces:

- `packages/plugins/frontend/core/panel/src/main.mesh`
- `packages/plugins/frontend/core/quick-settings/src/main.mesh`
- `packages/plugins/frontend/core/quick-settings/src/components/audio-section.mesh`
- `packages/plugins/frontend/core/quick-settings/src/components/wifi-section.mesh`
- `packages/plugins/frontend/core/quick-settings/src/components/wifi-item.mesh`

The panel is a compact status and entry surface. Quick settings is the primary control surface for audio and Wi-Fi.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none - native `.mesh` elements only |
| Icon library | XDG icon names rendered through MESH `<icon>` |
| Font | `token(typography.family)` / Inter |
| Theme source | `config/themes/mesh-default-dark.json` and `config/themes/mesh-default-light.json` |

### Non-Negotiable Design Rules

- Use theme tokens only: `token(color.*)`, `token(spacing.*)`, `token(radius.*)`, and `token(typography.*)`.
- Do not introduce hex colors in `.mesh` files.
- Do not introduce gradients, decorative blobs, oversized hero styling, marketing copy, or nested card layouts.
- Use `border-radius: token(radius.sm)` for framed controls and section surfaces unless an existing hover state already uses `token(radius.full)`.
- Keep letter spacing at `0`; do not add negative tracking or uppercase letter-spacing treatments in Phase 4 changes.
- Prefer icon-only buttons for mute/volume/network actions where the icon is self-evident, with `title` and `aria-label` set.

## Surface Contracts

### Top Panel

The top panel must remain compact and scannable.

| Element | Contract |
|---------|----------|
| Layout | Three zones: left workspace, centered clock, right service indicators. |
| Height behavior | Preserve current compact bar behavior; no expanded panel content. |
| Data display | Show at least one real backend service value; audio percent, audio icon, network status, and battery text are acceptable indicators. |
| Interaction | Volume/audio indicator may open quick settings. Direct panel controls are not required. |
| Error fallback | Use short neutral values such as `N/A`, `0`, or `disconnected`; detailed failures stay in diagnostics/logs. |

### Quick Settings Root

Quick settings must feel like an operational shell control surface, not a settings app.

| Element | Contract |
|---------|----------|
| Header | Keep title `Quick Settings`; close action remains a compact button labeled `Close`. |
| Section navigation | Left rail with icon plus short label for `Wi-Fi`, `Bluetooth`, and `Audio`. |
| Primary sections | Phase 4 requirements apply to `Wi-Fi` and `Audio`; Bluetooth remains out of scope unless touched only for layout consistency. |
| Drawer | One active section at a time. Empty drawer text stays short: `Select a section`. |
| Density | Controls must fit without overlapping at the existing drawer width; rows use fixed control dimensions for buttons and flexible width for sliders/lists. |

## Audio Section Contract

Audio must provide full primary controls.

| Element | Contract |
|---------|----------|
| Section title | `Audio` |
| Backend/source label | Show `audio.source_plugin` when present; otherwise `Unavailable`. |
| Percent value | Show integer percent as `{N}%`, width-constrained so it does not shift the row. |
| Volume slider | Range `0` to `100`, value from `audio.percent`, `onchange` calls the finalized proxy command path `audio.set_volume(...)`. |
| Step controls | Keep volume down and volume up buttons where supported. Labels may remain `-` and `+`; titles must be `Volume down` and `Volume up`. |
| Mute control | Icon button using `audio.toggle_mute()` where supported. Use `audio-volume-muted` when muted or zero. |
| Disabled state | If audio lookup fails, `audio.available == false`, or controls are denied, disable controls if the element supports disabled semantics; otherwise guard handlers and show disabled styling. |

### Audio Copy

| State | Copy |
|-------|------|
| Available | `Volume {N}%` |
| Muted | `Volume muted at {N}%` |
| Unavailable label | `Audio unavailable` |
| Permission denied label | `Audio controls unavailable` |
| Backend missing | `Unavailable` |

## Wi-Fi Section Contract

Network must provide core Wi-Fi controls, not a full network manager suite.

| Element | Contract |
|---------|----------|
| Section title | `Wi-Fi Networks` |
| Wi-Fi state | Read from `network.wifi_enabled`; expose a clear enabled/disabled state in the section. |
| Toggle | Use `network.set_wifi_enabled(...)` for Wi-Fi on/off. |
| Network list | Render `network.networks` when present and non-empty. |
| Network row | Each row shows network name and strength. Connect/disconnect is allowed only when provider data supplies a reliable id and active state. |
| Conservative command behavior | If provider data is insufficient, rows may be display-only or guarded. Do not fake success. |
| Disabled state | If network lookup fails, `network.available == false`, or control is denied, show visible disabled state and block command handlers. |

### Wi-Fi Copy

| State | Copy |
|-------|------|
| Enabled, list present | `Wi-Fi Networks` section with rows |
| Enabled, no list yet | `Scanning for networks` |
| Disabled | `Wi-Fi is disabled` |
| Unavailable | `Network unavailable` |
| Permission denied | `Network controls unavailable` |
| Unsafe connect data | `Connection details unavailable` |

## Spacing Scale

Declared values must use existing tokens and remain multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| `spacing.xs` | 4px | Icon gaps, label stacks, compact row internals |
| `spacing.sm` | 8px | Button padding, list item gaps, panel indicator gaps |
| `spacing.md` | 16px | Default row gaps, drawer padding, section gaps |
| `spacing.lg` | 24px | Quick settings outer padding |
| `spacing.xl` | 32px | Only for larger future layout gaps; avoid in Phase 4 controls unless needed |
| `spacing.2xl` | 40px | Not required for Phase 4 |
| `spacing.3xl` | 48px | Not required for Phase 4 |
| `spacing.4xl` | 64px | Not required for Phase 4 |

Exceptions: existing `gap: 2px` label stacks may remain for tight section subtitles. New spacing should use tokens.

### Stable Dimensions

| Element | Contract |
|---------|----------|
| Top panel icon | 16px or existing 18px for nav/volume icon contexts. |
| Quick settings nav rail | Preserve fixed rail width near `96px`. |
| Icon action buttons | 36px by 36px for quick-settings controls. |
| Volume percent | Fixed width near `56px` to avoid row shift. |
| Volume slider | Flexible width, fixed height near `36px`. |
| Network rows | Full drawer width, stable vertical padding. |

## Typography

Use existing theme typography. Do not scale font size with viewport width.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `token(typography.size.md)` / 14px | 400 | `token(typography.line-height.normal)` / 1.4 |
| Label | `token(typography.size.xs)` / 11px or `token(typography.size.sm)` / 12px | 500-700 | 1.4 |
| Heading | `token(typography.size.lg)` / 16px | 700 | 1.4 |
| Numeric value | `token(typography.size.lg)` / 16px | 800 | 1.4 |
| Panel indicator | `token(typography.size.sm)` / 12px | 400-600 | 1.4 |

Typography constraints:

- Letter spacing must be `0`.
- Avoid all-caps transforms for new text. Existing section labels may remain, but new Phase 4 text should use normal case.
- Long values must use constrained widths, wrapping, or ellipsis so they do not overlap adjacent controls.

## Color

Use semantic theme tokens, not raw hex values in implementation.

| Role | Dark Token / Value | Light Token / Value | Usage |
|------|--------------------|---------------------|-------|
| Dominant (60%) | `color.surface` / `#1C1B1F` | `color.surface` / `#FFFBFE` | Panel and quick settings base background |
| Secondary (30%) | `color.surface-container` / `#211F26` | `color.surface-container` / `#F3EDF7` | Drawer, section surfaces, buttons |
| Elevated secondary | `color.surface-container-high` / `#2B2930` | `color.surface-container-high` / `#ECE6F0` | Rows, hover surfaces, list items |
| Accent (10%) | `color.primary-container` / `#4F378B` | `color.primary-container` / `#EADDFF` | Active nav item, hover/selected control background |
| Accent foreground | `color.on-primary-container` / `#EADDFF` | `color.on-primary-container` / `#21005D` | Text/icons on active accent containers |
| Warning | `color.warning-container` / `#6A4300` | `color.warning-container` / `#FFE1A8` | Permission-denied or unavailable callouts only if needed |
| Disabled | `color.disabled` / `#938F99` | `color.disabled` / `#CAC4D0` | Disabled controls and unavailable values |
| Destructive | `color.error-container` / `#8C1D18` | `color.error-container` / `#F9DEDC` | Not expected in Phase 4; reserve for destructive confirmations only |

Accent reserved for: active section state, focused/hovered actionable controls, selected list rows, and slider active color. Do not color every interactive element with accent by default.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable; this is an operational shell surface, not a CTA flow. |
| Empty drawer heading | Not used. |
| Empty drawer body | `Select a section` |
| Audio unavailable | `Audio unavailable` |
| Audio permission denied | `Audio controls unavailable` |
| Network unavailable | `Network unavailable` |
| Network permission denied | `Network controls unavailable` |
| Wi-Fi disabled | `Wi-Fi is disabled` |
| Wi-Fi scanning | `Scanning for networks` |
| Connection unavailable | `Connection details unavailable` |
| Destructive confirmation | Not applicable; Phase 4 has no destructive actions. |

Copy constraints:

- Keep on-screen messages short enough for compact shell surfaces.
- Technical provider errors belong in diagnostics/logs, not primary UI copy.
- Button `title` and `aria-label` values must name the action: `Volume down`, `Volume up`, `Toggle mute`, `Toggle Wi-Fi`, `Connect to Wi-Fi network`, `Close quick settings`.

## Interaction Contract

| Interaction | Required Behavior |
|-------------|-------------------|
| Panel volume indicator click | Opens or toggles quick settings/audio controls; does not need to mutate audio state. |
| Quick settings close | Publishes the existing close event. |
| Audio slider change | Updates local displayed slider state optimistically only within valid range, calls `audio.set_volume(...)`, and relies on provider emit for canonical state. |
| Volume up/down | Calls `audio.volume_up()` and `audio.volume_down()` where available. |
| Mute | Calls `audio.toggle_mute()` where available. |
| Wi-Fi toggle | Calls `network.set_wifi_enabled(not network.wifi_enabled)`. |
| Wi-Fi row click | Calls `network.connect(id)` only when id is reliable and controls are available. |
| Missing service | Handler returns without command, visible disabled state remains. |
| Permission denied | Handler does not loop or fake success; visible disabled state or status copy remains. |

## Accessibility And Diagnostics

| Area | Contract |
|------|----------|
| Labels | Every icon-only button must have `title` and `aria-label`. |
| State text | Audio and Wi-Fi sections must expose readable state text, not icon-only status. |
| Disabled state | Disabled/unavailable controls must have a visible text cue in the section. |
| Diagnostics | Raw provider errors, missing interface details, and permission denial detail stay in logs/diagnostics. |
| Focus | Existing focus behavior may remain; do not add custom focus styling unless supported by the element/style system. |
