<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| quick-settings -> network provider | Frontend command payload crosses into NetworkManager backend command handlers. |
| shell/backend command data -> audio provider | Service command payload can contain untrusted sound file paths. |
| audio provider -> host shell | Provider commands execute host `wpctl`, `pactl`, and `aplay`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-05-01 | Tampering | `networkmanager-network:on_command_connect` | mitigate | Consume `payload.connection_id` from the public contract so connect/disconnect commands use the intended id. |
| T-04-05-02 | Denial of Service | `networkmanager-network:on_command_connect` | mitigate | Reject empty ids with `missing connection_id` before running host commands. |
| T-04-05-03 | Tampering | audio providers `on_command_set_muted` | mitigate | Implement the public `set_muted(device_id, muted)` method in both bundled providers. |
| T-04-05-04 | Elevation of Privilege | audio providers `on_command_play_sound` | mitigate | Replace `mesh.exec_shell("aplay " .. payload.path)` with structured `mesh.exec("aplay", { path })` after validating `path`. |
| T-04-05-05 | Information Disclosure | provider command errors | accept | Existing provider errors emit concise command result fields; Phase 4 UI keeps technical detail in logs/diagnostics per D-13. |
</threat_model>
