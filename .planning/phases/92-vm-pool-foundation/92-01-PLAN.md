  <action>
Create `crates/core/runtime/scripting/src/pool.rs` with the following public surface and behavior:

1. Imports: `use mlua::Lua;`, `use std::cell::RefCell;`, `use std::thread::ThreadId;`.

2. Define `pub struct LuaVmPool` with private fields:
   - `slots: Vec<Option<Lua>>` — `None` indicates a slot whose VM is currently checked out.
   - `floor: usize` — minimum number of slots maintained at construction (4 per POOL-03).

3. Define `pub struct PooledVm` with private fields:
   - `lua: Option<Lua>` — wrapped in `Option` so `Drop` can `take()` and move it back to the pool without `ManuallyDrop`.
   - `slot_index: usize`
   - `owner_thread: ThreadId`

4. Implement `impl LuaVmPool`:
   - `pub fn new(floor: usize) -> mlua::Result<Self>` — constructs `floor` VMs, each via `let lua = Lua::new(); lua.sandbox(true)?;` then pushed as `Some(lua)` into `slots`. Store `floor` for diagnostics.
   - `pub fn len(&self) -> usize` — returns `self.slots.len()`.
   - `pub fn available(&self) -> usize` — returns count of `Some` entries in `slots`.
   - `pub fn checkout(&mut self) -> PooledVm`:
     - Find the first slot index `i` where `slots[i].is_some()`. Take the `Lua` with `slots[i].take()`.
     - If no slot is available, push a freshly sandboxed `Lua` as `None` into `slots` (so the index is reserved as checked-out), `let new_lua = Lua::new(); new_lua.sandbox(true).expect("sandbox init failed");`, then use that `Lua` and `slot_index = slots.len() - 1`. (Implementation note: push `None` first to reserve the index, then return the new Lua to the caller as the PooledVm. Never block; never return Result.)
     - Capture `owner_thread = std::thread::current().id()` and return `PooledVm { lua: Some(lua), slot_index: i, owner_thread }`.
   - `pub(crate) fn return_slot(&mut self, slot_index: usize, lua: Lua)` — `self.slots[slot_index] = Some(lua);`. Add a `// Phase 94 (ISO-03): call `Thread::reset()` and drop env_table before reinsertion.` comment marker.

5. Define a `thread_local!` static pool used by `Drop`:
   ```
   thread_local! {
       static POOL: RefCell<LuaVmPool> =
           RefCell::new(LuaVmPool::new(4).expect("pool init"));
   }
   ```
   Expose `pub fn checkout() -> PooledVm` that calls `POOL.with(|cell| cell.borrow_mut().checkout())`.

6. Implement `impl Drop for PooledVm`:
   - `assert_eq!(std::thread::current().id(), self.owner_thread, "PooledVm dropped on a different thread than checkout");`
   - If `let Some(lua) = self.lua.take()`, call `POOL.with(|cell| cell.borrow_mut().return_slot(self.slot_index, lua));`
   - Use `assert!` (not `debug_assert!`) per RESEARCH.md Open Question 2 so tests pass in any profile.

7. Provide a `PooledVm::lua(&self) -> &Lua` accessor returning `self.lua.as_ref().expect("PooledVm lua already taken")`.

8. Add `#[cfg(test)] mod tests { use super::*; ... }` at the bottom of the file containing the five tests enumerated in `<behavior>`. Tests construct fresh `LuaVmPool` instances directly (not the `thread_local!` static) where possible to keep tests isolated; the wrong-thread test uses the module-level `checkout()` so the drop runs against the thread-local pool.

Per CONTEXT.md and RESEARCH.md anti-patterns: do NOT add the `fnv` crate. Do NOT use `Arc<Mutex<LuaVmPool>>`. Do NOT call `sandbox(true)` after any other operation on a fresh `Lua`.
