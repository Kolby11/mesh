<objective>
Implement `ChunkCache` as a process-wide source-string cache keyed by FNV-1a 64-bit content hash. The cache stores `String` source bodies (not compiled `mlua::Function`s — that would be VM-bound and unsafe to share across VMs). FNV-1a constants match the existing inline implementation in `runtime_tree.rs` so no `fnv` crate is introduced.

Purpose: Foundation type for v1.17 — multi-instance modules (two components from the same `.mesh` file) will resolve source from this cache once Phase 94 wires it into checkout. Phase 92 ships the standalone type with tests.

Output: `chunk_cache.rs` source + `pub mod chunk_cache;` declaration in `lib.rs`, all unit-tested.

2. FNV-1a constants (copy verbatim from `crates/core/shell/src/shell/component/runtime_tree.rs`):


3. `pub fn fnv64(bytes: &[u8]) -> u64`:
   - Initialize `hash = FNV_OFFSET`.
   - For each byte: `hash ^= u64::from(byte); hash = hash.wrapping_mul(FNV_PRIME);`
   - Return `hash`. Unlike `runtime_tree.rs`, do NOT skip the zero value — `0` is a valid (if unlikely) cache key.

4. Process-wide static and accessor:
   ```
   static SOURCE_CACHE: OnceLock<Mutex<HashMap<u64, String>>> = OnceLock::new();

   fn cache() -> &'static Mutex<HashMap<u64, String>> {
       SOURCE_CACHE.get_or_init(Default::default)
   }
   ```

5. `pub struct ChunkCache;` — a zero-sized handle type. All methods are associated functions or take `&self`. The actual storage is the process-wide static; `ChunkCache` is a typed entry point so call sites read clearly (`ChunkCache::get_or_insert(...)`).

6. `impl ChunkCache`:
   - `pub fn new() -> Self { Self }` — for symmetry with `LuaVmPool::new(...)`.
   - `pub fn get_or_insert(source: &str) -> u64`:
     - Compute `key = fnv64(source.as_bytes())`.
     - Lock `cache()`; `entry(key).or_insert_with(|| source.to_string())`.
     - Return `key`.
   - `pub fn get(key: u64) -> Option<String>`:
     - Lock `cache()`; `map.get(&key).cloned()`.
   - `pub fn remove(key: u64) -> Option<String>`:
     - Lock `cache()`; `map.remove(&key)`. Document with a comment: `// Phase 95 (CACHE-03): hot-reload mtime watcher calls this to evict on source change.`
   - `pub fn len() -> usize` — lock + `map.len()`. Used by tests.

7. Anti-patterns to avoid (per RESEARCH.md):

   - Do NOT add the `fnv` crate dependency.
   - Do NOT use LRU eviction — explicit eviction only, per Phase 92 scope.
