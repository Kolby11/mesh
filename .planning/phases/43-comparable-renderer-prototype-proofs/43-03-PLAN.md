# Plan 43-03 - MESH-Owned Focused-Crate Prototype Evidence

<objective>
Produce PROTO-02 evidence by rendering the shared navigation bar and audio popover fixture from retained MESH-shaped data through a focused-crate prototype boundary: Taffy-style layout, Parley-style text evidence, AnyRender-style paint commands, and AccessKit-compatible accessibility mapping.
</objective>

<must_haves>
- D-01: Compare structural and behavioral parity, not pixel-perfect production fidelity.
- D-02: Structured layout/render output is acceptable.
- D-03: Cover hover, click, slider movement/release, and open-close behavior.
- D-04: Use the shared fixture from Plan 43-01.
- D-05: Preserve stable node identity and display-list-like output.
- D-09: Prefer Taffy, Parley, AnyRender, and AccessKit for focused proof evidence.
- D-10: Use common comparison headings.
</must_haves>

<threat_model>
- T-43-07 Losing retained identity through external crate storage: High. Mitigate by keeping MESH stable node IDs in every output record and mapping crate node IDs back to them.
- T-43-08 Text/layout evidence too shallow: Medium. Mitigate by requiring `taffy_layout` and `parley_text` sections in the evidence output.
- T-43-09 Paint proof not comparable to current display list: Medium. Mitigate by emitting `display_slot` values for Background, Border, Text, Icon, and Generic.
- T-43-10 Accessibility forgotten as nonvisual proof: Medium. Mitigate by requiring `accesskit_node_id` and role output for retained nodes.
</threat_model>
