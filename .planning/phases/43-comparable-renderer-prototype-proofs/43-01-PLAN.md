# Plan 43-01 - Shared Prototype Fixture and Harness Skeleton

<objective>
Create the isolated Phase 43 prototype workspace and shared scenario fixture so the Blitz and focused-crate paths consume the same surface slice, interaction sequence, and evidence schema. This plan establishes the PROTO-03 comparison input contract before either prototype path runs.
</objective>

<must_haves>
- D-01: Compare structural and behavioral parity, not pixel-perfect production fidelity.
- D-02: Permit structured layout/render output when screenshots exceed throwaway scope.
- D-03: Cover hover, click, slider movement/release, and audio popover open-close behavior.
- D-04: Use the same scenario fixture set for both prototypes.
- D-05: Keep retained MESH-shaped data available for the focused-crate path.
- D-06: Allow Blitz to use HTML/CSS-equivalent fixtures.
- D-08: Keep Winit/Blitz shell behavior throwaway-only.
- D-10: Preserve common comparison headings for downstream evidence.
</must_haves>

<threat_model>
- T-43-01 Fixture drift between paths: High. Mitigate by creating one shared `phase43-scenarios.json` consumed by both binaries.
- T-43-02 Workspace dependency pollution: Medium. Mitigate by placing prototype code under `.planning/prototypes/phase43/` with a nested empty `[workspace]` table and no root `Cargo.toml` edits.
- T-43-03 Shallow evidence schema: Medium. Mitigate by requiring scenario IDs, stable node IDs, interaction event IDs, display slots, and comparison headings in shared code.
</threat_model>
