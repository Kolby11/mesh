# Plan 11-04 - Navigation-Bar Proof, Docs, and Regression Coverage

## Objective

Apply the new keyboard contract to real shell surfaces, document it for authors, and close Phase 11 with regression coverage that proves buttons, sliders, inputs, and navigation-bar controls all behave as required.

## Design Truths

- Navigation-bar is the real shell-chrome proof surface for keyboard navigation and shortcuts.
- Audio popover is the real shipped slider proof surface.
- Input behavior belongs in shell tests and focused fixtures, not in forced navigation-bar UI changes.
- Docs must explain the real `:focus-visible` and shortcut contract so later phases do not plan against the old alias model.

<threat_model>
T-11-04: The runtime could be correct in isolation while shipped modules, docs, or regression coverage remain out of sync, producing false confidence and later regressions.
Mitigation: update the real navigation-bar and audio-popover modules, set a concrete keyboard mode, document the new author-facing contract, and add end-to-end shell tests that exercise buttons, sliders, inputs, and focused-surface shortcuts together.
</threat_model>
