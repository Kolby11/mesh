# Plan 12-04 - Shell Keyframe Integration and Runtime Diagnostics

## Objective

Attach validated keyframe animation playback to live shell components, preserving animation timelines across reactive rebuilds, requesting frames only while needed, and surfacing runtime diagnostics for unresolved names or invalid animation references.

## Design Truths

- Stable `_mesh_key` plus animation name is the keyframe identity.
- Rebuilds do not restart a still-present animation.
- Completed finite animations stop dirty repaint churn.
- Runtime diagnostics are required for unresolved animation names and token/style failures that cannot be fully known at parse time.

<threat_model>
T-12-04: Active keyframe state could leak, restart every frame, or hide invalid animation references, causing either visible flicker or unnecessary redraw churn.
Mitigation: retain active animations only for live keyed nodes, preserve timelines by `_mesh_key` plus animation name, prune missing animations, and add shell diagnostics tests for unresolved animation names.
</threat_model>
