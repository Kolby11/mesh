# Plan 12-01 - Animation Token Namespace and Strict Token Resolution

## Objective

Establish the author-facing animation token contract: hard-rename theme motion tokens to `animation.*`, separate primitive animation values from default animation recipes, and make invalid animation token references diagnostic failures.

## Design Truths

- The canonical namespace is `animation.*`; `motion.*` is not preserved as a first-class alias.
- Primitive tokens such as `animation.duration.fast` and `animation.curves.bezier.one` are distinct from recipe tokens such as `animation.default.border-radius`.
- Default animation recipes compose primitives with explicit `token(...)` references.
- Invalid animation token references must fail visibly instead of producing silent default animation behavior.

<threat_model>
T-12-01: A silent `motion.*` compatibility path or fallback token resolution could hide author mistakes and make runtime animation behavior depend on stale theme names.
Mitigation: hard-rename shipped tokens, add grep-verifiable docs/config checks for `animation.*`, and add resolver diagnostics that reject invalid `token(animation.*)` references.
</threat_model>

## Tasks

<task id="12-01-01" wave="1">
<title>Hard-rename shipped theme motion tokens to animation tokens</title>
<read_first>
- `.planning/phases/12-theme-animation-tokens-and-css-animations/12-CONTEXT.md`
- `.planning/phases/12-theme-animation-tokens-and-css-animations/12-RESEARCH.md`
- `.planning/phases/12-theme-animation-tokens-and-css-animations/12-PATTERNS.md`
- `config/themes/mesh-default-dark.json`
- `config/themes/mesh-default-light.json`
- `docs/theming/themes.md`
</read_first>
<steps>
1. In `config/themes/mesh-default-dark.json` and `config/themes/mesh-default-light.json`, replace every `motion.duration.*` token with `animation.duration.*`.
2. Replace every `motion.easing.*` token with `animation.curves.bezier.*`, using exact keys such as `animation.curves.bezier.standard`, `animation.curves.bezier.standard-decelerate`, `animation.curves.bezier.standard-accelerate`, and `animation.curves.bezier.emphasized`.
3. Replace `motion.scale.hover`, `motion.scale.press`, and `motion.opacity.disabled` with `animation.scale.hover`, `animation.scale.press`, and `animation.opacity.disabled`.
4. Replace `motion.default.*` with `animation.default.*`.
5. Rewrite each `animation.default.*` recipe so any duration or curve component uses explicit `token(...)`, for example `animation.default.border-radius` must contain `border-radius token(animation.duration.fast) token(animation.curves.bezier.standard)`.
6. Update `docs/theming/themes.md` so the token group list names `animation`, not `motion`, and the token file examples include `animation.duration.fast`, `animation.curves.bezier.standard`, and `animation.default.border-radius`.
</steps>
<acceptance_criteria>
- `grep -R '"motion\\.' config/themes/mesh-default-dark.json config/themes/mesh-default-light.json` returns no matches.
- `config/themes/mesh-default-dark.json` contains `"animation.duration.fast"` and `"animation.curves.bezier.standard"`.
- `config/themes/mesh-default-light.json` contains `"animation.default.border-radius"`.
- `config/themes/mesh-default-dark.json` contains `token(animation.duration.fast)` in at least one `animation.default.*` value.
- `docs/theming/themes.md` contains `animation.duration.fast` and does not contain `motion,` in the token group list.
</acceptance_criteria>
<verify automated="true">
`grep -R '"motion\\.' config/themes/mesh-default-dark.json config/themes/mesh-default-light.json; test $? -eq 1 && grep -R "animation.duration.fast\\|animation.curves.bezier.standard\\|animation.default.border-radius" config/themes docs/theming/themes.md`
</verify>
</task>

<steps>
1. In `crates/core/ui/elements/src/style/resolve.rs`, add an animation-specific resolution path for `transition`, `transition-duration`, `transition-delay`, `transition-timing-function`, `animation`, `animation-duration`, `animation-delay`, and `animation-timing-function` that records a `StyleDiagnostic` when `token(animation.*)` cannot be resolved.
2. Ensure the invalid animation token path prevents that declaration from mutating `ComputedStyle.transition` or `ComputedStyle.animation`.
3. Preserve existing successful resolution for `token(animation.duration.fast)` as a numeric duration and `token(animation.curves.bezier.standard)` as a cubic-bezier easing string.
4. Add tests in `crates/core/ui/elements/src/style.rs` named with `animation_token` that prove valid animation tokens resolve and `token(animation.duration.fastest)` produces exactly one diagnostic containing `animation.duration.fastest`.
5. Keep non-animation token behavior unchanged for normal properties such as `background: token(color.surface)`.
</steps>

- A test proves `token(animation.duration.fastest)` produces a diagnostic message containing `animation.duration.fastest`.
