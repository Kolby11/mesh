# Plan 14-03 - Real Stage Instrumentation Across Input and Render Paths

## Objective

Instrument the actual shell and per-surface runtime seams that correspond to the milestone's required stage list so profiling reports real work rather than inferred approximations.

## Design Truths

- Build, restyle, and layout timing must be measured where those operations actually occur.
- Redraw count and total surface render time must be first-class metrics.
- Hidden surfaces should not pollute active summaries unless they performed real work.

<threat_model>
T-14-03: If stage hooks are added only around coarse render calls, the profiler will blur build/style/layout/paint boundaries, miss input/runtime work, and produce misleading hotspot data.
Mitigation: instrument `dispatch_wayland`, request/runtime update handling, `build_tree`, restyle, layout, paint, present, and outer per-surface render spans at the exact files where those stages execute today.
</threat_model>
