# Plan 14-02 - Bounded Profiling Collector and Session Storage

## Objective

Add a shell-owned profiling collector with fixed-count retention, rolling aggregates, and clean enable/reset semantics so the runtime can accumulate stable data without unbounded memory or UI coupling.

## Design Truths

- Collection must be fully bounded by fixed sample counts.
- The collector must preserve shell-wide work separately from per-surface work.
- Session reset behavior must be explicit and deterministic.

<threat_model>
T-14-02: A naïve collector can grow without bound, mix shell-global and per-surface work, or leak stale samples across profiling sessions, which makes the milestone data untrustworthy.
Mitigation: introduce a dedicated runtime profiling helper with fixed-capacity rings, explicit shell vs surface buckets, and enable-time session reset behavior covered by tests.
</threat_model>
