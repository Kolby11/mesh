# Plan 14-04 - Snapshot Rollups and Regression Proof

## Objective

Convert collected profiling state into stable debug snapshots and finish the phase with automated regression coverage that proves Phase 14's required data model and timing hooks work without expanding into inspector UI scope.

## Design Truths

- Snapshot assembly must stay centralized in `build_debug_snapshot()`.
- The snapshot must be stable enough for later live inspector views without changing the runtime model again.
- Tests, not manual overlay inspection, are the required proof for this phase.

<threat_model>
T-14-04: Even with correct instrumentation, the milestone can fail if snapshot assembly drops required metrics, hides surface identity, or leaves disabled-mode semantics unproven.
Mitigation: extend `build_debug_snapshot()` to emit stable shell-wide and per-surface profiling summaries, then add regression tests covering disabled-mode silence, reset semantics, stage coverage, and surface accounting.
</threat_model>
