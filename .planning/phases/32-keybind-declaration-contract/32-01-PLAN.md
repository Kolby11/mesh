# Plan 32-01: Keybind Declaration Contract and Compatibility Bridge

<objective>
Add a typed keybind declaration model for frontend modules and bridge the current navigation-bar `keyboard.shortcuts` behavior through that model without changing locale resolution, conflict diagnostics, or global shortcut behavior yet.
</objective>

<requirements>
- `KEYB-01`: Frontend modules can declare semantic keybind actions with stable action ids.
- `KEYB-02`: Each keybind action can define handler, target control reference, scope, label/i18n key, and default trigger metadata.
- `KEYB-03`: Manifest/settings parsing validates keybind declarations into typed Rust structures instead of relying on ad hoc JSON at dispatch time.
</requirements>

<must_haves>
- A normalized manifest type exposes typed keybind actions keyed by stable action id.
- JSON, TOML, and package-style manifest loading preserve keybind declarations in the normalized manifest.
- Existing `settings_json.keyboard.shortcuts.mute` dispatch still works for `@mesh/navigation-bar`.
- Existing `KeyboardSettings.surface_shortcuts` override behavior still works by shortcut/action id.
- Invalid or missing keybind declarations do not break modules without keybinds.
</must_haves>

<threat_model>
No new external input source is added; module manifests and settings are already untrusted module-authored data. Main risks are malformed declarations causing crashes, shortcut hijacking of existing shell/global keys, and user override drift. Phase 32 mitigates by parsing into typed structures, keeping shell-global shortcut precedence unchanged, preserving user override keys by action id, and treating absent keybinds as empty declarations.
</threat_model>

<action>
Add typed keybind declarations to the normalized manifest model.

In `crates/core/extension/module/src/manifest/model.rs`:
- Add `#[serde(default)] pub keybinds: KeybindsSection,` to `Manifest`.
- Add these public structs/enums:
  - `KeybindsSection { #[serde(default)] pub actions: HashMap<String, KeybindAction> }`
  - `KeybindAction { pub handler: String, #[serde(default)] pub target_ref: Option<String>, #[serde(default)] pub scope: KeybindScope, #[serde(default)] pub label: Option<String>, #[serde(default)] pub label_i18n_key: Option<String>, #[serde(default)] pub trigger: KeybindTrigger }`
  - `KeybindTrigger { #[serde(default)] pub kind: KeybindTriggerKind, #[serde(default)] pub key: Option<String>, #[serde(default)] pub modifiers: Vec<String> }`
  - `KeybindScope` with serde `rename_all = "snake_case"` variants `Surface` default and `Access`.
  - `KeybindTriggerKind` with serde `rename_all = "snake_case"` variants `Shortcut` default and `AccessKey`.
- Add helper methods:
  - `impl KeybindsSection { pub fn is_empty(&self) -> bool }`
  - `impl KeybindAction { pub fn validate(&self, action_id: &str) -> Result<(), String> }`
- Validation must reject empty `handler`, reject `trigger.key == None` or blank key, and reject blank `target_ref`, `label`, or `label_i18n_key` when those options are present.
- Add `impl Manifest { pub fn validate_keybinds(&self) -> Result<(), String> }` that iterates `self.keybinds.actions` and calls `KeybindAction::validate(action_id)`.

In `json.rs`, `toml.rs`, and `package/module_manifest.rs`:
- Add pass-through support so normalized `Manifest.keybinds` is populated for regular `module.json`, legacy TOML manifests, and package-style manifests.
- In `package/module_manifest.rs`, add `#[serde(default)] pub keybinds: manifest::KeybindsSection` to `MeshModuleSection`, include `keybinds: mesh.keybinds` in `into_runtime_manifest`, and include `keybinds: manifest.keybinds.clone()` in `from_legacy_manifest`.
- In `MeshModuleSection::validate`, call `self.keybinds.validate()` or equivalent validation before `self.contributes.validate()`.

In `manifest/tests.rs`:
- Add a test named `parses_module_json_keybind_declarations`.
- The test JSON must include:
  - top-level `"keybinds": { "actions": { "accept": { "handler": "onAccept", "target_ref": "accept-button", "scope": "surface", "label_i18n_key": "action.accept", "trigger": { "kind": "shortcut", "key": "Enter", "modifiers": ["ctrl"] } } } }`
- Assert the parsed manifest has action id `accept`, handler `onAccept`, target ref `accept-button`, scope `Surface`, label_i18n_key `action.accept`, trigger kind `Shortcut`, key `Enter`, and modifiers `["ctrl"]`.
- Add a test named `module_json_without_keybinds_has_empty_keybinds`.
- Add a test named `package_json_keybinds_round_trip_to_runtime_manifest` using package-style `"mesh": { "keybinds": { "actions": { ... } } }` and assert `load_manifest(...).manifest.keybinds.actions` contains the action.
- Add a test named `invalid_keybind_declaration_is_rejected` that proves a blank handler or missing trigger key fails validation through the package-style manifest path.
</action>

- `modules/frontend/navigation-bar/config/settings.json`

<action>
Replace direct ad hoc runtime shortcut parsing in `FrontendSurfaceComponent::resolved_surface_shortcuts` with a typed compatibility bridge.

In `crates/core/shell/src/shell/component/input/keyboard.rs`:
- Introduce an internal `SurfaceShortcutDeclaration` struct with fields:
  - `action_id: String`
  - `key: String`
  - `handler: String`
  - `target_ref: Option<String>`
- Add a function `surface_shortcut_declarations_from_settings(settings_json: &serde_json::Value) -> Vec<SurfaceShortcutDeclaration>`.
- This function must read `settings_json.keyboard.shortcuts`, use the map key as `action_id`, require non-empty `key` and `handler`, preserve optional `target_ref`, and ignore invalid entries.
- Change `ResolvedSurfaceShortcut` to include `action_id: String`.
- Change `resolved_surface_shortcuts` to iterate over `SurfaceShortcutDeclaration`, apply `keyboard_settings.surface_shortcuts[self.surface_id()][action_id].key` overrides, and return `ResolvedSurfaceShortcut` records.
- Do not change dispatch ordering in `input/mod.rs`: Tab, Escape, Ctrl+C, then module/surface shortcut dispatch must remain in that order.

In `crates/core/shell/src/shell/component/tests/interaction/navigation.rs`:
- Extend `keyboard_shortcuts_surface_handler_runs_and_metadata_matches_binding` or add a sibling test proving an override still uses the stable action id `mute`.
- The test must set `KeyboardSettings.surface_shortcuts["@mesh/navigation-bar"]["mute"].key = "u"` through the existing config/test mechanism or component hook available in tests, then prove pressing `u` dispatches while default `m` does not.
</action>
