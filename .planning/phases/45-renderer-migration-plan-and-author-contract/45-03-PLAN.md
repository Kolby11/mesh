# Plan 45-03 - `.mesh` Renderer Author Contract

<objective>
Create the author-facing renderer contract for plugin-authored `.mesh` UI and link it from existing authoring docs so future renderer migration does not overpromise browser semantics or hide stability guarantees.
</objective>

<must_haves>
- D-08: No broad `.mesh` behavior change in Phase 45.
- D-09: Document retained identity, theme-owned selection, diagnostics/profiling, and AccessKit-compatible direction.
- D-10: Document unsupported browser-like expectations.
- D-11: Document stable authored surface behavior during migration.
- D-16: Keep audio transition polish deferred.
- D-17: Keep module install requirement resolution separate.
- D-18: Cite existing Blitz dependency evidence instead of reopening research.
</must_haves>

<threat_model>
- T-45-03-01 Browser compatibility overpromise: High. Mitigate by including exact non-goal strings for HTML, Blitz, Winit, DOM, and web platform behavior.
- T-45-03-02 Public API leak: High. Mitigate by stating proof snapshots and crate-facing conversion modules are not public author APIs.
- T-45-03-03 Author docs not discoverable: Medium. Mitigate by linking the renderer contract from `docs/frontend/mesh-syntax.md` and `docs/module-system.md`.
</threat_model>
