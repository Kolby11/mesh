# Plan 45-02 - Renderer Ownership Classification

<objective>
Create `docs/renderer-ownership.md` as the source-backed classification of existing renderer modules and candidate renderer boundaries as authoritative, adapter-owned, or replacement candidates.
</objective>

<must_haves>
- D-04: Classify current shell/render/presentation ownership as authoritative.
- D-05: Classify focused proof outputs and crate-facing conversion modules as adapter-owned.
- D-06: Classify candidate crates and future runtime expansions as replacement candidates or future adoption candidates.
- D-07: Keep Blitz as reference/blocker evidence.
- D-15: Tie authority changes to observability parity.
- D-18: Cite existing Blitz evidence without reopening research.
</must_haves>

<threat_model>
- T-45-02-01 Ownership drift: High. Mitigate by listing exact source paths for each authoritative and adapter-owned boundary.
- T-45-02-02 Candidate crates mistaken for public guarantees: High. Mitigate by putting Taffy, Parley, AnyRender/Vello-style rendering, AccessKit runtime expansion, Stylo, and Skia only under `replacement candidate`.
- T-45-02-03 Blitz status ambiguity: Medium. Mitigate by including `Blitz remains reference/blocker evidence, not a production authoring model.`
</threat_model>
