<objective>
Add the retained-layout foundation in `layout.rs`: a `PerSurfaceLayoutState` struct that holds a persistent `TaffyTree`, a `_mesh_key → TaffyNodeId` map, last available space, and a `valid` flag; a `remove_taffy_subtree` post-order removal helper; and a new `LayoutEngine::compute_incremental` entry point that routes the non-structural dirty paths (fresh-build fallback, VISUAL_REPAINT skip, LAYOUT mark_dirty+compute, available-space-change recompute).     - When state.valid == true and the LAYOUT bit is set: mark_dirty is called and compute_layout_with_measure runs, then write_taffy_layout updates WidgetNode.layout
    - When available size changed (w,h != state.last_available): compute_layout_with_measure runs even if no node bits are dirty, and state.last_available is updated
  </behavior>
  <action>
    Accept the dirty signal as a plain `u8`/`u16` bitmask parameter so `layout.rs` takes NO dependency on `mesh-core-shell`. Add to `LayoutEngine`:

    `pub fn compute_incremental(root: &mut WidgetNode, state: &mut PerSurfaceLayoutState, available_width: f32, available_height: f32, dirty_layout: bool, dirty_structural: bool, intrinsic_cache: &mut IntrinsicLayoutCache, measurer: Option<&dyn TextMeasurer>)`.

    Here `dirty_layout` means "the LAYOUT bit was set" and `dirty_structural` means "a TREE_REBUILD (SCRIPT|TEXT) occurred". The shell-side mapping from ComponentDirtyFlags to these two booleans is done in plan 03. Keep the existing `compute_with_intrinsic_cache_and_measurer` and `compute_taffy_layout_with_cache` functions unchanged — `compute_incremental`'s fresh-build branch may delegate to a shared private helper or inline the same logic.

    Routing logic (in this order):
    1. If `!state.valid` OR `dirty_structural`: run the FRESH-BUILD path. Clear `state.tree` (assign `TaffyTree::new()`), build a fresh `node_map: HashMap<String, TaffyNodeId>` keyed by `_mesh_key`, build a fresh `text_nodes: HashMap<NodeId, TextMeasureData>`, call `build_taffy_tree`, run `compute_layout_with_measure`, `write_taffy_layout`, set `state.valid = true`, set `state.last_available = (available_width, available_height)`. Store the freshly built `_mesh_key → TaffyNodeId` map into `state.node_map`. (For now, `dirty_structural` routes here as a correct-but-non-incremental fallback; plan 02 replaces it with the structural diff.)
    2. Else if `(available_width, available_height) != state.last_available`: call `compute_layout_with_measure` on the retained tree's root, `write_taffy_layout`, update `state.last_available`. No per-node set_style/mark_dirty needed.
    3. Else if `dirty_layout`: walk the tree, for each node with a `_mesh_key` present in `state.node_map`, call `state.tree.mark_dirty(taffy_id)`. Then `compute_layout_with_measure` + `write_taffy_layout`. Do NOT update last_available (unchanged).
    4. Else (VISUAL_REPAINT only — neither structural nor layout nor size change): for each node with a `_mesh_key` in `state.node_map`, recompute its style via `taffy_style_for_node` and call `state.tree.set_style(taffy_id, style)`. Do NOT call compute_layout and do NOT call write_taffy_layout — prior-frame WidgetNode.layout values remain valid (RESEARCH.md Pitfall 1).

    Because `build_taffy_tree` keys its internal map by `NodeId`, add a small post-build step (or a parallel walk) that maps each node's `_mesh_key` to the corresponding `TaffyNodeId`. The simplest correct approach: after `build_taffy_tree` fills a `HashMap<NodeId, TaffyNodeId>`, walk `root` and for every node that has a `_mesh_key`, insert `(mesh_key.clone(), node_id_to_taffy[&node.id])` into `state.node_map`. Nodes without `_mesh_key` are not tracked in node_map (they are only reachable via the fresh-build path) — note this limitation in a code comment referencing RESEARCH.md Pitfall 3.

    Rebuild `text_nodes` on every pass that calls compute_layout (fresh-build, size-change, layout-dirty) because NodeId is ephemeral (RESEARCH.md Pitfall 5). The VISUAL_REPAINT path does not call compute_layout so it does not need text_nodes.

    Add `#[test]` cases: `compute_incremental_fresh_build_matches_baseline` (build a small tree, run compute_incremental with valid=false, capture rects; run compute_with_intrinsic_cache_and_measurer on an identical tree; assert per-node LayoutRect equality), and `compute_incremental_visual_repaint_preserves_layout` (run a valid pass, mutate a paint-only style, run compute_incremental with dirty_layout=false dirty_structural=false, assert layout rects unchanged from the prior pass).
