  <behavior>
    - Adding a child node (new _mesh_key in the new tree) creates a new TaffyNode, inserts it into node_map, and attaches it to its parent via set_children; sibling rects with unchanged keys are byte-equal to the pre-add rects where geometry did not shift
    - Removing a node (a _mesh_key in node_map absent from the new tree) calls remove_taffy_subtree and drops the key (and its descendant keys) from node_map; tree.total_node_count() reflects only surviving nodes
    - Reordering children updates the parent via set_children in the new order; resulting rects match a fresh rebuild of the reordered tree
    - A node whose _mesh_key matches an existing entry reuses its TaffyNodeId and calls set_style (no new node created)
    - After the structural pass, compute_layout_with_measure runs and write_taffy_layout populates WidgetNode.layout
  </behavior>
  <action>
    Replace the plan-01 placeholder body of the `dirty_structural` branch (which currently routes to fresh-build) with a reconciliation walk over `root` against `state.node_map`:

    1. Build a `present_keys: HashSet<String>` of every `_mesh_key` found in the new tree (depth-first).
    2. Walk the new tree top-down. For each node:
       - If it has a `_mesh_key`:
         - If the key is in `state.node_map`: reuse the existing TaffyNodeId; recompute style via `taffy_style_for_node` and call `state.tree.set_style(taffy_id, style)`. Record the (NodeId → TaffyNodeId) pairing for write_taffy_layout.
         - If the key is NOT in `state.node_map`: create the Taffy node (`new_leaf` if no children else create empty then attach later) with the computed style, insert `(_mesh_key.clone(), taffy_id)` into `state.node_map`, and record the pairing.
       - If it has NO `_mesh_key` (RESEARCH.md Pitfall 3): treat as always-new — create a fresh Taffy node each rebuild; it is not stored in node_map (only reachable this pass). Add a code comment citing Pitfall 3.
    3. For each parent node in the new tree, after its children's TaffyNodeIds are known, call `state.tree.set_children(parent_taffy_id, &child_taffy_ids_in_order)` to fix child set and order (handles both add and reorder).
    4. Removal: for every key in `state.node_map` that is NOT in `present_keys`, call `remove_taffy_subtree(&mut state.tree, taffy_id)` and remove the key from `state.node_map`. Do removals AFTER reparenting so removed nodes are already detached from surviving parents by set_children.
    5. Determine the root TaffyNodeId (the new tree root's mapped id) and call `compute_layout_with_measure` (rebuild text_nodes this pass since NodeId is ephemeral — RESEARCH.md Pitfall 5), then `write_taffy_layout(root, &state.tree, &node_id_to_taffy)`. Set `state.last_available = (available_width, available_height)` and `state.valid = true`.

    Build the `node_id_to_taffy: HashMap<NodeId, TaffyNodeId>` that `write_taffy_layout` requires during the same walk (NodeId of each WidgetNode → its TaffyNodeId), since `write_taffy_layout` is keyed by NodeId not _mesh_key.

    Keep the fresh-build path for `!state.valid` exactly as plan 01 left it — only the `state.valid && dirty_structural` case uses the new diff.
  </action>
