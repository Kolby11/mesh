<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-02-09: old poll loop keeps emitting after provider replacement | high | Store backend task handles by interface and abort/close old slot before inserting replacement. |
| T-02-10: old command receiver remains reachable after stop | high | Remove `service_handlers[interface]` before dropping/aborting the runtime slot. |
| T-02-11: failed init still leaves a sender in `service_handlers` | high | Register sender only after lifecycle startup path has validation and runtime slot ownership; remove on `InitFailed`. |
| T-02-12: restart behavior becomes automatic failure recovery | medium | Implement explicit replacement/stop primitive only; do not schedule automatic restart after runtime failure. |
</threat_model>
