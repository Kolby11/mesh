# Plan 10-01 - Input Contract and Selection Ownership

## Objective

Preserve modifier-aware key input, reconcile the narrowed Phase 10 requirement boundary, and add shell-owned selection lifecycle state that coexists cleanly with existing controls.

## Design Truths

- D-01: Selectability remains explicit opt-in via the existing text-node `selectable` field.
- D-05: A Phase 10 drag selection may span only one selectable text node; wrapped lines within that node are allowed.
- D-07: Clipped or ellipsized text is not part of Phase 10 selection or copy semantics.
- D-13: Standard copy shortcuts only act on Phase 10 selection when such a selection exists.

<threat_model>
T-10-01: Selection or copy routing could steal keyboard/pointer behavior from existing controls or preserve stale state after a rebuild.
Mitigation: Preserve modifier data in `ComponentInput`, keep selection state shell-owned and `_mesh_key`-validated, and add regression tests for control precedence and selection clearing.
</threat_model>

<task id="10-01-02" wave="1">
<title>Add shell-owned selection lifecycle state with control-safe pointer boundaries</title>
<read_first>
- `crates/core/shell/src/shell/component.rs`
- `crates/core/shell/src/shell/component/input.rs`

</read_first>
<steps>
1. Add selection state to `FrontendSurfaceComponent` near the existing focus/hover/pointer fields, keyed by stable `_mesh_key` identity rather than transient `WidgetNode::id`.
2. Start selection only when pointer-down lands on passive selectable text, not on buttons, sliders, switches, checkboxes, inputs, or shell chrome.
3. Define the Phase 10 clear rules: click elsewhere, control-focused keyboard intent, surface hide, or selected-node removal clears selection.
4. Add shell-level regression tests for pointer-down precedence, same-node clamping, and deterministic clearing after rebuild/hide.
</steps>
<acceptance_criteria>
- Interactive control text remains non-selectable and preserves existing pointer behavior.
- Selection lifecycle state survives repaint/rebuild when the selected node still exists and clears deterministically when it does not.
- No selection state is keyed by transient node IDs.
</acceptance_criteria>
