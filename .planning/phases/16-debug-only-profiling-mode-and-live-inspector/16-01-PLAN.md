# Plan 16-01 - Debug Inspector State Contract and Frontend API Surface

## Objective

Define the read-only debug/profiling contract that the `.mesh` inspector will consume, and expose explicit shell events for opening the overlay and toggling profiling without creating a separate diagnostics path.

## Design Truths

- The inspector must consume shell-owned debug state through a stable frontend-facing contract.
- Profiling remains opt-in and separate from overlay visibility.
- The new contract must speak in Phase 16 inspector terms, not the old native-tab vocabulary alone.

<threat_model>
T-16-01: Phase 16 can accidentally ship a private native-only data path or implicitly couple profiling to overlay visibility, which would violate the debug-only milestone rule and make the `.mesh` inspector non-reusable.
Mitigation: add a canonical `mesh.debug` contract, make the shell publish a read-only payload from `build_debug_snapshot`, and add explicit debug-control service events that mutate shell state without auto-enabling profiling.
</threat_model>
