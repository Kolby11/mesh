# Plan 16-02 - Shell-Shipped Debug Inspector Module and Overlay Mount Path

## Objective

Create the built-in debug inspector module/package and connect the existing overlay toggle to that `.mesh` surface so the native debug panel stops being the primary UI host.

## Design Truths

- The inspector must mount like a normal frontend surface, even though the shell ships it by default.
- Debug overlay toggles now control inspector surface visibility rather than native panel paint.
- The old native renderer can remain only for layout-bounds overlays or other non-panel responsibilities.

<threat_model>
T-16-02: The phase can leave the native panel half-alive while also adding a `.mesh` inspector, producing two divergent debug UIs and unclear ownership of overlay visibility.
Mitigation: add one canonical `@mesh/debug-inspector` surface, route overlay toggles to its visibility, and remove panel-content painting from the native debug overlay path.
</threat_model>

</steps>
