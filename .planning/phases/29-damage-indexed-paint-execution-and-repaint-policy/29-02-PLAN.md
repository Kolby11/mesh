# Plan 29-02 - Debug-inspector retained paint counter readability

## Objective

<objective>
Close the Phase 29 UAT observability gap by making retained paint filtering counters visible in the shipped debug-inspector Surfaces view, so operators can distinguish minimal-damage filtering from broad or full-surface repaint fallback without reading raw payload JSON.
</objective>

## Gap

- UAT Test 1 reported that the debug view was too messy to tell whether repainting was excessive.
- Root cause: Phase 29 serialized `invalidation.paint` counters, but the actual Surfaces tab only rendered timing and redraw labels.

## Tasks

<tasks>

<task id="29-02-01" wave="1">
<title>Render retained paint policy and filtering counters in the Surfaces view</title>
<action>
1. Add surface-row props and visible text rows for repaint policy, fallback count, filtered command count, filtered span count, and skipped command count.
2. Populate those props from `snapshot.invalidation.paint` in the debug-inspector module script.
3. Keep missing payloads explicit with unavailable labels rather than silently hiding the proof.
</action>
<acceptance_criteria>
- A surface row displays `Paint policy {policy}; fallbacks {count}`.
- A surface row displays `Filtered {commands} commands from {spans} spans; skipped {skipped}`.
- Existing surface timing, total render, and redraw labels remain visible.
</acceptance_criteria>
