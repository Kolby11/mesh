# Plan 17-03: Inspector Benchmark UI Rows

<objective>
Replace the scaffold-only benchmark cards with compact runnable scenario rows that consume `mesh.debug.benchmarks` safely and show latest-session metrics inside the existing debug inspector.
</objective>

<requirements>
- `BENCH-01`: Hover row displays input/restyle/render metrics.
- `BENCH-02`: Surface open/close row displays render/redraw metrics.
- `BENCH-03`: Pointer-driven update row displays input-to-visible-response metrics.
- `BENCH-04`: Keyboard traversal row displays focus/input/render metrics.
- `BENCH-05`: Backend-driven update row displays backend and frontend render metrics.
</requirements>

<decision_coverage>
- `D-03`: scaffold-only cards become runnable scenario rows/controls inside the existing inspector.
- `D-05`: rows use shipped surface anchors: `@mesh/navigation-bar`, `@mesh/audio-popover`, and `@mesh/debug-inspector`.
- `D-12`: empty, skipped, and unavailable scenario states remain visible.
- `D-13`: automated proof uses existing Rust shell/component tests, not a new E2E harness.
- `D-15`: real-surface component tests cover the benchmark view after it becomes runnable.
</decision_coverage>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Malformed debug payload crashes the inspector | high | Parent `.mesh` code must type-check tables and provide string fallbacks for every row prop. |
| Benchmark UI hides unavailable scenarios and makes suite appear incomplete | medium | Component tests must assert all five titles render when profiling is off and when payload is sparse. |
| New UI violates compact debug-tool design | medium | Follow `17-UI-SPEC.md`: row layout, theme tokens only, no nested cards, no hard-coded colors. |
</threat_model>
