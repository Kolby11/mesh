# Plan 09-02 - Size and Container Query Invalidation

## Objective

Make surface and container size changes re-evaluate container queries and relayout the tree without forcing plugin reloads or losing runtime state.

## Design Truths

- D-01: Surface size and child container-size changes are first-class style invalidation triggers; container queries use current width/height before layout and paint.
- D-02: Size-driven and state-driven restyles may rebuild or re-resolve the tree but must keep existing frontend runtime state.
- D-03: Invalidation happens explicitly at the component boundary in `FrontendSurfaceComponent::paint`, `build_tree`, and surface-size dispatch.
- D-14: Include at least one real `FrontendSurfaceComponent` regression because runtime preservation during rebuild/restyle is the main risk.

<threat_model>
T-09-02: A size change could reuse stale layout/style or trigger a runtime reset.
Mitigation: Track rendered dimensions as an explicit invalidation input and rebuild/restyle with existing runtime state.
</threat_model>
