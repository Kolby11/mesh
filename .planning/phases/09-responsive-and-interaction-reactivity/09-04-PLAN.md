# Plan 09-04 - State Preservation and Cleanup

## Objective

Preserve user/runtime state through size and interaction restyles, while clearing state for nodes that are removed or hidden by the final tree.

## Design Truths

- D-09: Restyles preserve input values, slider values, checked values, scroll offsets, hover/focus/active identities where valid, service payloads, settings, locale/theme state, and imported runtime state.
- D-10: If a restyle removes or hides the previous hovered/focused/active node, the corresponding state clears deterministically with existing event semantics.
- D-12: State-driven style transitions use existing transition metadata and style animation plumbing only where it already applies; custom keyframes remain Phase 12.
- D-15: Navigation-bar can be referenced as a future proof target, but Phase 13 owns comprehensive navigation-bar behavior.

<threat_model>
T-09-04: Restyle cleanup could either leak stale interaction targets or erase valid user/service state.
Mitigation: Preserve state maps by stable key and prune only keys proven absent from the final tree.
</threat_model>
