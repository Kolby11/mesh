<task type="auto">

  <files>crates/core/runtime/backend/src/lib.rs</files>
  <read_first>crates/core/runtime/backend/src/lib.rs, crates/core/runtime/scripting/src/backend.rs, .planning/phases/02-backend-lifecycle-foundation/02-CONTEXT.md</read_first>
  <action>Add `const MAX_CONSECUTIVE_POLL_FAILURES: u32 = 3`. In `spawn_backend_service()`, keep script load before `call_init()`. If `load_script()` fails, emit `Failed { stage: "load", ... }` and return. If `call_init()` fails, emit `InitFailed { ... }` and return before creating the poll interval/select loop. After successful init, emit `Started`. During polling, increment a consecutive failure counter on `run_poll()` error, emit `PollFailed { count, message }`, and when count reaches 3 emit `Failed { stage: "poll", ... }`, then `Stopped`, and exit. Reset the counter after a successful poll. Preserve `refresh_interval()` after successful poll and command paths.</action>
