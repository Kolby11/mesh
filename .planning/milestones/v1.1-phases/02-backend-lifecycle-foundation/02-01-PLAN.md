| T-02-01: invalid backend manifests launch anyway | high | Validate module kind, provided interface, readable entrypoint, and binary availability before creating channels or runtime tasks. |
| T-02-02: shell silently falls back to another provider when explicit provider fails | high | Use only `InstalledModuleGraph::active_provider()` for normal Phase 2 startup; do not call `fallback_provider()` in graph-driven startup. |
| T-02-03: disabled backend modules create stale runtimes | high | Resolver excludes disabled nodes and tests assert disabled active providers produce no launch candidate. |
