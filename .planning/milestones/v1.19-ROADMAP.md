## Key Accomplishments

- Shell event loop replaced `std::thread::sleep` with blocking `poll()` on Wayland connection fd using deadline from `next_runtime_sleep()` — eliminates idle CPU burn
- `eventfd` IPC wakeup added so backend and IPC message sends immediately wake the blocking poll, preventing stale-frame latency on service events
- Dev-window backend preserved with existing `std::thread::sleep` path — `supports_blocking_dispatch()` gate on `PresentationEngine`
- `ProfilingStage::SchedulerIdle` captures block duration and wake reason (WaylandEvent / IpcEvent / DeadlineExpired) for debug inspector
- `wl_surface::set_opaque_region` sent from present path using conservative root-background-only opaque rect computation from retained display list
- Zero new crate dependencies — all work uses existing `rustix::event::eventfd`/`poll`, `smithay-client-toolkit`, and `wayland-client`
