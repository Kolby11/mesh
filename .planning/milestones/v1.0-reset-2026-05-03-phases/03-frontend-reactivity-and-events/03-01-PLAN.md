<objective>
Make frontend reactive global dirty semantics explicit and testable, then wire the shell so handler-triggered rebuilds follow script state changes instead of unconditional handler calls.

Purpose: Phase 3 depends on a precise contract: changed globals rebuild UI; unchanged globals do not; the explicit redraw escape hatch still works.
Output: Runtime and shell dirty-state behavior with regression tests for changed globals, unchanged globals, shallow tables, redraw requests, and next-paint rebuild.
</objective>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| A handler that assigns the same value causes a rebuild every time, making continuous events noisy and unpredictable | medium | Keep dirty marking value-based and add tests where repeated assignment to the same scalar leaves `ScriptState::is_dirty()` false after clearing. |
| Removing unconditional shell dirty marking suppresses legitimate UI updates after handlers mutate globals | high | After every successful handler call, check the target runtime's `script_ctx.state().is_dirty()` and set `FrontendSurfaceComponent.dirty = true` only when script state dirtied or shell-owned input state changed. |
| Table globals accidentally rely on deep diff behavior that is not part of the public contract | medium | Implement an explicit shallow table comparison helper and cover added, removed, changed, unchanged, and nested-only table cases. |
| Scripts lose their ability to force a render when no tracked global changed | medium | Preserve `__mesh_request_redraw = true`, clear it after sync, and add a regression test proving it sets dirty even when no user global changed. |
</threat_model>
