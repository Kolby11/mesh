<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Structured `mesh.exec(program, args)` accidentally routes through a shell and enables argument injection | high | Implement structured exec with `StdCommand::new(program).args(args)` only; keep shell execution exclusively behind `mesh.exec_shell(command)`. |
| `mesh.config()` leaks another plugin's settings | medium | Store settings per `BackendScriptContext` instance and add a test that reads only the provided settings table. |
| Bad `mesh.service.emit()` payloads silently reuse stale payloads | medium | Reset pending emit before each handler and add a test proving invalid emits do not publish old state. |
| Unknown log levels panic backend scripts | low | Normalize levels and fall back to warn/debug behavior without panicking. |
</threat_model>
