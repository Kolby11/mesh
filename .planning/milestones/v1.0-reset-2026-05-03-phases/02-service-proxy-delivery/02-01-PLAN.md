<objective>
Make the frontend proxy runtime honor the new Phase 02 semantics: explicit diagnostics on lookup failure, field-level dependency tracking, command-only proxy methods, and value-based dirty invalidation driven by backend emissions.

Purpose: The phase cannot be planned or executed coherently until the runtime stops treating service callbacks as part of the public model and starts invalidating only the rendered service fields that actually changed.
Output: Updated runtime/shell proxy plumbing plus regression coverage for field tracking, rerender invalidation, and visible diagnostics.
</objective>

| Threat | Severity | Mitigation |

| `pcall(require, ...)` hides missing or invalid service contracts and leaves the UI silently degraded | high | Emit a plugin-visible error diagnostic before returning `ScriptError::InterfaceUnavailable(...)` or `ScriptError::CapabilityDenied(...)`, including plugin id, interface name, and reason. |
| Whole-service invalidation rerenders every consumer on every emit and prevents the field-level model from matching the locked Phase 02 decisions | high | Track top-level field reads inside `create_service_proxy(...)`, snapshot tracked field values, and only mark a component dirty when one of those values actually changes. |
| Legacy `bind/on/on_change` semantics remain exposed and keep the runtime split between two different reactivity models | medium | Remove public proxy exposure for `bind` and `on_change`, stop relying on `mesh.service.on(...)` in invalidation flow, and prove rerender behavior with field-driven tests. |
| Proxy command channel routing regresses while the proxy surface is tightened | low | Keep `mesh.<service>.<command>` publishing and `script_events_to_requests(...)` translation intact with regression coverage. |
</threat_model>
