Rework the interface and provider contract layer so Phase 02 has a portable base interface, formally declared provider inheritance, command-only proxy methods, and runtime-authoritative state fields that shipped surfaces can read directly.

Purpose: The locked Phase 02 decisions require a real base interface plugin model, but they also reject contract metadata as the sole truth for rich provider state. This plan establishes the base/core contract while keeping richer provider extras additive and runtime-driven.
Output: Updated manifest/contract/runtime metadata, aligned core interface packages, and provider payload shapes that support read-fields / command-methods semantics.
</objective>
