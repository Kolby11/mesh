# Phase 50 Plan 01: AccessKit Runtime Update Adapter And Adoption Gates

## Objective

Build the retained-node AccessKit runtime update boundary behind `renderer-accesskit`, prove it with feature-enabled tests, and close the renderer-library adoption documentation gates.

## Tasks

1. Add `accesskit_adapter.rs`
   - Feature-gated with `#![cfg(feature = "renderer-accesskit")]`.
   - Convert `WidgetNode` recursively to `accesskit::TreeUpdate`.
   - Map MESH roles to AccessKit roles.
   - Preserve retained `NodeId` as `accesskit::NodeId`.
   - Set label/description/value/bounds/children/control actions/state where available.

2. Wire render crate exports
   - Register the private module in `lib.rs`.
   - Re-export `build_accesskit_runtime_update` only under `renderer-accesskit`.

3. Update adoption docs
   - `docs/renderer-migration.md`: mark AccessKit retained-node update evidence complete and record final v1.9 gates.
   - `docs/renderer-ownership.md`: classify AccessKit retained-node runtime updates as adapter-owned production boundary; keep platform publication as replacement candidate/deferred.
   - `docs/frontend/renderer-contract.md`: clarify author-facing accessibility metadata remains stable while AccessKit update construction becomes internal runtime direction.

## Verification

- `cargo check -p mesh-core-render`
- `cargo test -p mesh-core-render --features renderer-accesskit accesskit`
- `cargo test -p mesh-core-render --features renderer-libraries renderer_library`
- `cargo test -p mesh-core-shell phase44_navigation`
- Documentation grep for AccessKit/adoption classifications.
