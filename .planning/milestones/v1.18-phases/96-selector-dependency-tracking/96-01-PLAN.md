<objective>
Build SEL-01 and SEL-02: augment `StyleRuleIndex` with a reverse index (`state_to_rules`) that maps individual state bits (e.g., `STATE_HOVERED = 1 << 0`) to the rule indices that depend on that state, and expose `rules_for_state_bit()` for O(1) lookup.

Purpose: Per D-01, the existing `state: Vec<(u32, Vec<usize>)>` stores combined bitmask→rules entries. When a single state bit changes (e.g., `:hover` is removed), we need to know which rules depend on `STATE_HOVERED` specifically — not which rules depend on any combination that includes `STATE_HOVERED`. The reverse index `state_to_rules` stores per-bit rule lists so that future invalidation (Phase 98) can efficiently determine which rules are affected by a state-bit change.

Output: `StyleRuleIndex` struct gains `state_to_rules: HashMap<u32, Vec<usize>>`. `index_state_selector()` populates it alongside the existing `state` Vec. `rules_for_state_bit(bit: u32) -> &[usize]` provides O(1) lookups. Unit tests verify correctness.
</objective>

    use crate::style::{StyleRule, StyleDeclaration, StyleValue, Selector};

                value: StyleValue::Literal("red".to_string()),
