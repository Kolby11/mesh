<objective>
Plumb the eventfd file descriptor (created by Plan 03 in `Shell::run()`) into the IPC server and backend spawn paths. After every `tx.send()` or `shell_tx.send()` that delivers a `ShellMessage` to the shell's message channel, immediately write `1` to the eventfd. This ensures that the blocking `poll()` in the shell loop (Plan 03) is woken whenever IPC commands or backend state updates arrive, preventing stale-frame latency.

Purpose: Complete the IPC wakeup mechanism (SCHED-02). Without this plan, the shell would only wake on Wayland frame callbacks or deadline expiry — IPC commands and backend state changes would sit in the channel until the next compositor event.
Output: Eventfd is signaled on every IPC and backend message send, waking the blocking poll in the shell loop.
</objective>
