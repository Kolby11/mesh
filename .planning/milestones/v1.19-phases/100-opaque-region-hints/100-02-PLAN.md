**Part A: Add opaque rect computation helper function**

Add a private function in `render.rs` (inside the existing `impl Shell` block, or as a free function in the file — whichever fits the codebase patterns):

```rust
fn compute_opaque_rect_for_root(commands: &[DisplayPaintCommand], surface_width: u32, surface_height: u32) -> Option<DamageRect>
```

**Implementation logic (per CONTEXT.md decisions D-02 and D-03):**

1. **Get root command:** The root node's paint command is always `commands[0]` (every non-hidden WidgetNode produces a Node command as its first entry). If `commands.is_empty()`, return `None`.

2. **Access root style:** `let root_style = &commands[0].node.style;`

3. **Guard condition 1 — background_color.a:** `if root_style.background_color.a != 255` → return `None`. MUST use exact equality 255, NOT `>= 255` or `== u8::MAX`. Per PITFALLS.md Pitfall 3: any alpha < 1.0 (255) means the compositor MUST blend background content — marking as opaque causes rendering artifacts.

4. **Guard condition 2 — background_paint:** `if root_style.background_paint != BackgroundPaint::None` → return `None`. `BackgroundPaint::None` means a solid color fill (the `background_color`). `Image(...)` and `LinearGradient(...)` may have partially-transparent regions even if the root's overall background_color.a is 255.

5. **Guard condition 3 — border-radius:** `if root_style.border_radius > 0.0` → return `None`. Rounded corners mean parts of the surface are transparent at the edges. The opaque region must be a rectangle — it can't describe rounded corners, so skip entirely.

   Test: `border_radius > 0.0` — use `> 0.0`, not `> 0` (it's f32). The EPSILON comparison is not needed here — theme tokens set border-radius to exact values (0, 4, 8, 16, 24) and the CSS parser produces these exactly.

6. **Guard condition 4 — overflow clipping:** Check `root_style.overflow_x.clips_contents() && root_style.overflow_y.clips_contents()`. Both must clip. If either dimension is not `Hidden` (i.e., `Visible`, `Scroll`, or `Auto`), child content may extend beyond the root's bounds, and the compositor would incorrectly skip rendering background content behind non-clipped children. Only `Overflow::Hidden` returns true from `clips_contents()`.

   The condition: `if !root_style.overflow_x.clips_contents() || !root_style.overflow_y.clips_contents()` → return `None`

7. **All guards passed:** Return `Some(DamageRect { x: 0, y: 0, width: surface_width.max(1), height: surface_height.max(1) })`.

   **Why full surface:** The opaque region for a solid-color root background with no border-radius covers the entire surface. Using `surface_width x surface_height` rather than the root node's layout bounds ensures the region matches the actual Wayland buffer dimensions (which may include padding/margins beyond the root widget).
