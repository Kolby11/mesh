## Overview

v1.20 wires three Wayland compositor protocols to reduce CPU work and support HiDPI displays:
per-region damage tracking so unchanged pixels are never re-uploaded, HiDPI / fractional scale
so surfaces render at native physical density with `wp_viewporter` for non-integer ratios, and
KDE compositor blur offload so `backdrop-filter: blur(...)` surfaces delegate blur to the compositor
instead of blurring client pixels. Phase 103.1 was inserted to close three gaps found during the
milestone audit before archiving.

## Milestone Summary

**Decimal Phases:**
- Phase 103.1: Audit Gap Closure (inserted after Phase 103 to close DMGE-03, CR-01, and CR-02 before archiving)

**Key Decisions:**
- Use `org_kde_kwin_blur` (wayland-protocols-plasma 0.3) not `wp_blur_v1` — no shipping compositor implements the latter
- No CPU software blur fallback — clients cannot read the compositor framebuffer; unsupported compositors render flat background
- Damage rects capped at 16 per frame to bound protocol overhead
- Phase order: damage → HiDPI → blur (scale must be authoritative before blur region coordinates are correct)
- `protocol_damage_rects` helper extracted as pure function for testability
- SHM copy region stays unioned; per-rect `damage_buffer` only for protocol calls
- `wp_viewporter` lives in wayland-protocols::wp::viewporter, not wayland-protocols-wlr
- `fractional_scale` requires staging feature flag on wayland-protocols
- Scale factor handlers clamp values: 1..=3 (integer) and 60..=480 (fractional) per threat model

**Issues Resolved:**
- CR-01: Blur region not cleared on backdrop-filter removal — fixed by `blur_committed: bool` guard in Phase 103.1
- CR-02: Negative layout coordinate saturation in compute_blur_region — fixed with `max(0.0) as u32` clamping in Phase 103.1
- DMGE-03: `damage_rect_count` reporting binary 0/1 instead of actual count — path confirmed correct via `effective_damage.damage_rect_count()`

**Issues Deferred:**
- Per-surface damage rect count telemetry in the production monitoring path — debug only for v1.20
- Widget-level opaque rect analysis (OPAQUE-02) — carried from v1.19
- WR-01 (Phase 103): Blur region cleared before hide() — partially addressed in Phase 103.1
- `ext_background_effect_v1` blur protocol support — deferred until compositor adoption confirmed

**Technical Debt Incurred:**
- Phase 101 VALIDATION.md remains a partial stub (nyquist_compliant: false) — existing tests cover DMGE-01/02 but formal validation doc was not filled in
