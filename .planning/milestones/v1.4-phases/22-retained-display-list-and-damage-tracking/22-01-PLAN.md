# Plan 22-01: Retained Display List and Damage Metrics

## Goal

Add a retained display-list tracker and rectangle damage summary derived from the current widget/render-object data, then expose honest retained-paint metrics through existing debug profiling snapshots without changing visual output.

## Tasks

### 22-01-01: Render-Crate Display List Types

- Add focused display-list and damage types under `crates/core/frontend/render/src/`.
- Key retained entries by `NodeId` plus primitive slot/type.
- Compute rectangle damage from old and new entry bounds, clipped to the surface.
- Track reused, rebuilt, removed, damage area, full-surface fallback, partial-present support, and skipped-paint pixels.
- Add unit tests for reuse, changed bounds, removal, clipping, and full-surface fallback.

### 22-01-02: Component Integration and Debug Snapshot

- Store a retained display-list tracker on `FrontendSurfaceComponent`.
- Update it after retained render-object synchronization and before painting.
- Extend `ProfilingInvalidationSnapshot` with retained paint/display-list metrics.
- Include the new metrics in debug JSON so `mesh.debug.profiling.surfaces[].invalidation` exposes Phase 22 counters.

### 22-01-03: Verification and Documentation

- Run focused tests for render display-list and shell debug snapshot paths.
- Record summary and verification artifacts.
- Mark Phase 22 complete only if tests pass and visual-output code paths remain full-buffer compatible where partial present is unsupported.

## Verification

- `cargo test -p mesh-core-render display_list`
- `cargo test -p mesh-core-shell profiling`
- `cargo fmt --check`

## Risks

- Current presentation backends may not support partial present. Metrics must report `partial_present_supported=false` and `skipped_paint_pixels=0` rather than implying unsupported work was skipped.
- Dirty worktree contains prior Phase 19-21 changes. Phase 22 edits must preserve those changes and avoid broad renderer rewrites.
