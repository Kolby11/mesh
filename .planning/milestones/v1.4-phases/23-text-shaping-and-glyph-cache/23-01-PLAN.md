# Plan 23-01: Text Layout Cache and Debug Metrics

## Goal

Add retained text shaping/layout cache behavior to `TextRenderer`, reuse cached layout buffers across measure/render/selection operations, and expose text cache metrics through existing debug profiling snapshots.

## Tasks

### 23-01-01: Text Renderer Cache

- Add a bounded text layout cache inside `TextEngine`.
- Key cached layouts by text content, font family, font size, font weight, line height, wrapping width, and alignment.
- Reuse cached `cosmic_text::Buffer` layouts for measurement, rendering, and selection geometry.
- Track hits, misses, invalidations/evictions, shaped entries, and glyph cache availability.

### 23-01-02: Profiling Integration

- Add text cache metrics to `mesh_core_debug` profiling invalidation snapshots.
- Return text cache metrics from frontend paint calls without disrupting existing callers.
- Attach text metrics to `FrontendSurfaceComponent` invalidation snapshots.
- Serialize metrics in debug JSON under the existing surface invalidation payload.

### 23-01-03: Verification

- Add focused unit tests for cache hit/miss behavior and invalidation on text input changes.
- Extend shell profiling tests to prove text metrics are carried.
- Run formatting and focused render/shell tests.

## Verification

- `cargo fmt --check`
- `cargo test -p mesh-core-render text_cache`
- `cargo test -p mesh-core-render`
- `nix develop -c cargo test -p mesh-core-shell profiling_snapshot_exposes_typed_surface_invalidation_counts`

## Risks

- `cosmic_text::Buffer` reuse must not hold stale text inputs. Cache keys must cover all shaping inputs used by the current renderer.
- Selection highlight repaint should not destroy reusable shaping, but selection-specific paint still needs correct overlay clipping.
