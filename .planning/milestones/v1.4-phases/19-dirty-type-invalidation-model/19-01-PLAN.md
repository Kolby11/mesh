# Plan 19-01: Expose typed dirty invalidation in debug profiling

## Goal

Make dirty invalidation observable as structured profiling data while preserving the existing retained fast path and full-rebuild fallback.

## Tasks

- [x] Add debug data types for component dirty categories and retained widget-tree dirty categories.
- [x] Record the last invalidation snapshot per profiled surface.
- [x] Serialize invalidation counts into `mesh.debug.profiling.surfaces[].invalidation`.
- [x] Keep script/text mutations routed to full rebuild and style/layout/paint/surface changes routed to the retained path when valid.
- [x] Verify with formatting and focused shell tests.

## Acceptance

- Debug profiling exposes script/state/style/layout/paint/text/accessibility/metrics/surface-config counts.
- Debug profiling exposes retained inserted/removed/layout/style/attributes/children/state counts.
- Full rebuild and retained-path booleans are visible for the last profiled surface render.
- Existing typed invalidation tests continue to pass.
