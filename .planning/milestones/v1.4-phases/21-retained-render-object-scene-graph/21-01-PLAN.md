# Plan 21-01: Synchronize stable widget nodes into retained render objects

## Goal

Create a retained render-object tree that tracks slot-level changes from stable widget nodes.

## Tasks

- [x] Add `RenderObjectTree` to `mesh-core-render`.
- [x] Split render-object snapshots into transform, clip, opacity, geometry, material, text, and accessibility slots.
- [x] Report inserted, removed, reordered, and mutated slot counts through `RenderObjectDirtySummary`.
- [x] Synchronize render objects from `FrontendSurfaceComponent` after widget/style finalization.
- [x] Preserve existing software painter behavior.

## Acceptance

- Render objects are keyed from stable widget node IDs.
- Slot-specific dirty summaries distinguish geometry/material/text/etc. changes.
- Shell paint updates the retained render-object tree without changing visual output.
