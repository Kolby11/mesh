# Plan 20-01: Target retained interaction restyles by stable node key

## Goal

Reduce retained-path style propagation for interaction-state changes by restyling only nodes whose previous or current state is non-default.

## Tasks

- [x] Add keyed restyle support to `StyleResolver`.
- [x] Collect previous and current stateful node keys around runtime annotation.
- [x] Use keyed restyle only for retained interaction-state invalidations.
- [x] Reuse retained layout rectangles when targeted interaction restyles do not change layout-relevant style inputs.
- [x] Fall back to full layout when targeted restyles change layout-relevant style inputs.
- [x] Preserve the conservative full restyle path for rebuild, text, script, and non-interaction style/layout changes.
- [x] Add focused tests for keyed restyle and retained invalidation routing.

## Acceptance

- Hover/focus retained restyles recompute only affected stable nodes.
- Nodes leaving hover/focus are recomputed from base rules, removing stale state styles.
- Non-layout interaction style changes keep retained layout output.
- Layout-affecting interaction style changes retain the full-layout correctness fallback.
- Existing typed invalidation routing continues to pass.
