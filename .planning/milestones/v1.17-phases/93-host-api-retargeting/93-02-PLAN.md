    1. Check the mlua import line at the top of the file. If `Table` is not already imported from mlua, add it. The import line likely looks like `use mlua::{..., LuaValue, ...};` — add `Table` to the list.

    2. Change the function signature at line 466 from:
       `fn install_host_api(&mut self) -> mlua::Result<()>`
       to:
       `fn install_host_api(&mut self, target: &mlua::Table) -> mlua::Result<()>`

    3. Change line 467 from:
       `let globals = self.lua.globals();`
       to:
       `let globals = target;`
       This single rename makes every `globals.set(...)` call throughout the function body use `target` automatically.

    4. Update the call site in `new()` at line 133. The current code is:
       ```rust

           .lua
           .globals()
           ...
       ```

       ```rust
       let globals = ctx.lua.globals();

           .expect("backend host API setup should succeed");
       ```
       The `ctx.builtin_globals = ctx.lua.globals()...` lines that follow remain unchanged.
