# Plan 13-02 - Control Cluster Enrichment and Dormant Component Reuse

## Objective

Keep the control cluster compact and keyboard-stable while deciding which dormant local components should be mounted or adapted so the shipped nav bar looks richer without crossing into deferred feature work.

## Design Truths

- Existing controls stay button-first and keyboard-stable.
- Reuse of dormant local components is preferred over inventing new surface widgets.
- Reusing the existing battery-status widget is allowed; expanding it into a popover/stats feature is not.

<threat_model>
T-13-02: The richer proof could quietly become a new product-feature phase by turning dormant components into new interactive surfaces or by altering control semantics during layout work.
Mitigation: reuse only existing visible status/control behavior, preserve current button handlers and service wiring, and document the module inventory after migration.
</threat_model>
