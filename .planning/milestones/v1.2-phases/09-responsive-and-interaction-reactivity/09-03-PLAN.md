# Plan 09-03 - Post-Restyle Synchronization

## Objective

Guarantee that layout bounds, hit testing, accessibility data, published metrics, refs, and rendered output all observe the final post-restyle tree.

## Design Truths

- D-04: Any layout-affecting restyle is followed by layout recomputation before hit testing, accessibility metric publishing, and paint.
- D-11: `refs` and `elements` metrics update after layout so scripts read fresh bounds on the next render tick.
- D-13: Tests cover hit-test synchronization, accessibility/metric synchronization, pseudo-state changes, and container query changes.

<threat_model>
T-09-03: Stale bounds after a restyle could route clicks or accessibility metadata to the wrong element.
Mitigation: Recompute layout after the final restyle and store/publish only that tree.
</threat_model>
