# Plan 09-01 - Stable Interaction State Restyle

## Objective

Ensure hover, focus, active, disabled, checked, and focus-visible pseudo styles are applied from stable runtime keys during rebuild/restyle, without relying on transient `NodeId`.

## Design Truths

- D-05: The source of truth for interaction state is shell-owned stable `_mesh_key` / path-based runtime state, not transient `NodeId`.
- D-06: `mesh-core-elements::InputState` remains a lower-level reference/test utility unless this phase introduces stable IDs there.
- D-07: Hover, focus, active, disabled, checked, and focus-visible must map into `ElementState` before final style resolution.
- D-08: Hover updates only when hit path/key changes; focus persists until explicit transfer, surface reset, or later keyboard traversal work.

<threat_model>
T-09-01: Incorrect state identity could style or activate the wrong node after a rebuild.
Mitigation: Source pseudo state from stable `_mesh_key` paths and clear stale targets only after final tree validation.
</threat_model>
