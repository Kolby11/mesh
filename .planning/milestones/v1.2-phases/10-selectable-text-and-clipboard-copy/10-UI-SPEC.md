# Phase 10 - UI Design Contract

> Visual and interaction contract for Phase 10 selectable text and clipboard copy. Generated by `gsd-ui-phase`, verified inline against the six UI-SPEC dimensions.

---

## Scope

Phase 10 introduces a narrow, first-release proof for selectable rendered text in MESH.

The approved UI proof is **not** the navigation bar, audio slider, button chrome, or a general shell-wide selection behavior. The proof surface is a **dedicated read-only fixture** whose only job is to demonstrate:

- passive shell copy at rest
- mouse drag text selection
- theme-owned selected-state highlight
- standard copy-shortcut expectations

This UI contract intentionally excludes broad `TEXT-04` presentation claims from the current milestone docs. The UI proof must not imply support for clipped text, nested multi-node selection, or interactive-control selection.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none - native `.mesh` elements only |
| Icon library | XDG icon names rendered through MESH `<icon>` where needed, though Phase 10 proof should not require iconography |
| Font | `token(typography.family)` / Inter |
| Theme source | `config/themes/mesh-default-dark.json` and active shell theme tokens |

### Non-Negotiable Design Rules

- Use theme tokens only: `token(color.*)`, `token(spacing.*)`, `token(radius.*)`, and `token(typography.*)`.
- Do not introduce raw hex colors in `.mesh` implementation files.
- Do not add hover hints, helper badges, copy icons, “copyable” labels, or instructional affordances to selectable text at rest.
- Do not restyle top-level shell chrome just to advertise Phase 10 behavior.
- Do not use action-oriented copy or branded promo copy for the proof fixture. Treat it as passive system/read-only text.
- The selected state fully overrides the text block’s ordinary foreground/background styling while selected.

## Surface Contracts

### Dedicated Proof Fixture

The first approved proof surface is a small read-only text fixture created specifically for Phase 10.

| Element | Contract |
|---------|----------|
| Purpose | Prove opt-in text selection and copy behavior safely, without coupling Phase 10 to existing control surfaces. |
| Interactivity | No buttons, sliders, switches, inputs, or clickable action rows inside the proof fixture. |
| Text structure | One selectable text node is the primary proof target. The fixture may include adjacent non-selectable framing copy, but the proof range itself stays within one selectable node. |
| Layout | Compact shell-card or section style using existing surface tokens; no hero layout, no full-page composition, no decorative marketing treatment. |
| Clipping | The selectable proof text must remain fully visible. Clipped or ellipsized text is out of scope for the proof fixture. |
| Overflow | Wrapped multi-line text is allowed and should be used if needed to prove wrapped-range behavior inside a single node. |

### Existing Shell Surfaces

| Surface | Contract |
|---------|----------|
| Navigation bar | Remains visually unchanged for Phase 10 UI proof. No selection affordance or proof obligation on top-level bar chrome. |
| Audio popover | Not the primary proof surface for this phase. Existing passive copy may inform later work, but Phase 10 UI approval does not depend on it. |
| Interactive controls | Button labels, slider labels, switch labels, and other actionable control text remain non-selectable in this phase. |

## Selection Proof Contract

### Rest State

At rest, the proof text must look like ordinary passive shell copy.

| Element | Contract |
|---------|----------|
| Idle styling | No extra hint, underline, cursor-only trick, helper text, or icon announcing copyability. |
| Copy tone | Read-only system/status prose rather than branded, promotional, or CTA-style copy. |
| Visual weight | Match existing shell text density and restraint; selectable text should not look “special” until selection exists. |

### Drag And Selected State

| Element | Contract |
|---------|----------|
| Drag feedback | Highlight appears only during drag or after a selection exists. |
| Highlight ownership | Highlight styling is shell/theme-owned, not component-authored per fixture. |
| Text override | Selected text uses selection foreground/background colors even if the base text uses different passive colors. |
| Boundary clarity | Highlight should stop cleanly at the selection boundary inside the selectable node. |
| Non-selectable neighbors | Nearby framing text or container chrome should stay visually unaffected. |

### Copy State

| Element | Contract |
|---------|----------|
| Shortcut assumption | UI assumes standard explicit copy shortcut behavior only. |
| After copy | Selection may remain visible after successful copy. |
| No alternate copy model | No copy-on-select, no primary-selection, no middle-click paste signaling in the UI. |

### Clear State

| Element | Contract |
|---------|----------|
| Clear behavior | Selection may clear when intent moves elsewhere, but the UI should not add extra helper messaging or animated exit treatment. |
| Visual transition | Selection disappearance should feel immediate and functional, not theatrical. |

## Spacing Scale

Declared values must use existing tokens and remain multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| `spacing.xs` | 4px | Tight inline gaps only if the proof fixture needs micro-spacing |
| `spacing.sm` | 8px | Compact text stack spacing |
| `spacing.md` | 16px | Default padding and text-block spacing for the fixture |
| `spacing.lg` | 24px | Outer section or card padding if the proof fixture needs a larger shell surface |
| `spacing.xl` | 32px | Not required for the default proof fixture |
| `spacing.2xl` | 40px | Not required |
| `spacing.3xl` | 48px | Not required |
| `spacing.4xl` | 64px | Not required |

Exceptions: none

### Stable Dimensions

| Element | Contract |
|---------|----------|
| Proof fixture width | Should fit comfortably inside a compact shell surface without requiring ellipsis or horizontal clipping. |
| Text block padding | Default to `spacing.md`; may use `spacing.sm` only for especially compact proof layouts. |
| Radius | Use `token(radius.lg)` for a standalone proof card or `token(radius.md)` / `token(radius.sm)` when embedded in an existing shell section. |

## Typography

Use the existing theme typography. Do not scale text by viewport or try to create a document-reader aesthetic.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | `token(typography.size.md)` / 14px | 400 | `token(typography.line-height.normal)` / 1.4 |
| Secondary label | `token(typography.size.sm)` / 12px | 400-500 | 1.4 |
| Heading | `token(typography.size.sm)` / 12px or `token(typography.size.md)` / 14px | 500-700 | 1.4 |
| Selected proof text | Same base size as body text; selected state changes color, not type scale or weight | Preserve base weight | Preserve base line height |

Typography constraints:

- Keep letter spacing at `0`.
- Do not introduce uppercase tracking or editorial display styling.
- Wrapped text must remain readable and calm in a shell context.
- Selected state should not change font size, weight, or line height.

## Color

Use semantic theme tokens, not raw hex values in implementation.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `color.surface` / `#1C1B1F` | Surrounding shell background |
| Secondary (30%) | `color.surface-container` / `#211F26` | Proof fixture surface or card background |
| Accent (10%) | `color.primary` / `#6750A4` or theme-equivalent accent family | Selection background emphasis |
| Accent foreground | `color.on-primary` / `#FFFFFF` or theme-equivalent readable foreground | Selected text color |
| Destructive | `color.error` / `#F2B8B5` | Not used by the Phase 10 proof fixture |

Accent reserved for: the selected text range and only the selected text range in the proof fixture. Do not spread the accent family across all passive text or surrounding shell chrome.

### Selection Tokens

Phase 10 introduces these shell-owned tokens:

| Token | Default Direction | Usage |
|-------|-------------------|-------|
| `color.selection-background` | Accent-led; derived from the theme’s primary/accent family | Background for selected text |
| `color.selection-foreground` | Strong readable foreground against the selection background | Foreground for selected text |

Token constraints:

- `color.selection-background` should feel clearly distinct from ordinary surface containers.
- `color.selection-foreground` must remain highly readable against the chosen selection background.
- The selection pair should read clearly on the default dark theme without requiring extra outlines or idle affordances.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable; this phase is a passive interaction proof, not a CTA flow. |
| Empty state heading | Not applicable |
| Empty state body | Not applicable |
| Error state | Use short, factual problem copy only if the proof fixture has an unavailable state; otherwise not required for Phase 10 UI proof |
| Destructive confirmation | Not applicable |
| Proof text tone | Passive system/read-only shell copy with no action request |

Copy constraints:

- The proof fixture text should sound like shell status or explanatory copy, not like a button label or product marketing.
- Do not add “Drag to copy,” “Copy this,” or similar instructional copy to the fixture by default.
- Avoid technical implementation jargon in the visible proof text.

---

## Interaction Contract

| Interaction | Required Behavior |
|-------------|-------------------|
| Pointer down on selectable text | Begins the selection interaction without changing idle shell chrome elsewhere. |
| Drag across selectable text | Produces a clear selected range within the one selectable text node. |
| Drag outside selectable text | Selection may clamp at the last valid character; UI should not suggest broader selection support. |
| Drag across wrapped lines | Highlight remains visually coherent within the wrapped text block. |
| Copy shortcut | UI assumes the standard explicit copy shortcut path only. |
| Click elsewhere | May clear selection without additional messaging, badges, or tutorial affordances. |
| Interactive controls nearby | Must retain normal button/slider/input behavior and must not appear selectable. |

---

## Accessibility And Diagnostics

| Area | Contract |
|------|----------|
| Readability | Passive and selected text must remain readable on the active theme. |
| State change | Selection is communicated visually through the highlight, not through extra helper labels. |
| Control separation | Interactive controls near the proof fixture must remain visually and behaviorally distinct from selectable passive text. |
| Diagnostics | Internal selection/copy failures belong in diagnostics/logging, not in persistent visible UI copy by default. |
| Focus styling | Phase 10 does not introduce a new focus treatment through this UI contract; keyboard-specific selection behavior is deferred. |
