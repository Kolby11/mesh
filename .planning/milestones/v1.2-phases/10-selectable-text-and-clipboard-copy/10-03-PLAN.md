# Plan 10-03 - Clipboard Routing and Dedicated Proof Fixture

## Objective

Add explicit clipboard write routing for selected text, then prove the feature on a dedicated passive fixture that matches the approved UI contract and locks in boundary regressions.

## Design Truths

- D-04: Phase 10 proof lives on a dedicated read-only fixture, not on top-level interactive shell chrome.
- D-11: Selection visuals remain theme-owned, not component-authored.
- D-14: A successful copy keeps the visible selection intact.
- D-16: No copy-on-select, primary-selection, or middle-click paste behavior ships in this phase.

<threat_model>
T-10-03: Clipboard routing could leak hidden text, bypass normal control behavior, or prove the feature on an interactive surface that reintroduces rejected scope.
Mitigation: Use explicit shell-owned write-only clipboard plumbing, copy only the visible selected substring, and add a dedicated passive proof fixture with regression tests for control boundaries.
</threat_model>
