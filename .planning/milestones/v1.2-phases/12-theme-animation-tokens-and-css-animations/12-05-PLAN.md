# Plan 12-05 - Author Documentation and Phase Coverage Proof

## Objective

Finish Phase 12 by aligning docs, examples, and focused proof tests with the implemented animation contract, making sure every ANIM requirement and locked context decision is visibly covered before execution moves to Phase 13.

## Design Truths

- Documentation must use `animation.*`, not `motion.*`.
- Examples use explicit `token(...)` references.
- Keyframes are percentage-only and strict.
- Phase 12 proof should stay focused; full navigation-bar migration belongs to Phase 13.

## Decision Coverage

- **D-01:** The docs and final proof use `animation.*` as the canonical namespace.
- **D-02:** The final docs and config checks prove `motion.*` has been hard-renamed and is not preserved as a long-term alias.
- **D-03:** The final docs describe flat animation tokens, not nested structured preset objects.
- **D-04:** The final docs distinguish primitive animation token values from shell default animation recipes.
- **D-05:** The final docs include primitive examples such as `animation.duration.fast` and `animation.curves.bezier.one`.
- **D-06:** The final docs include default recipe examples such as `animation.default.border-radius`.
- **D-07:** The final docs show default recipes composed with explicit `token(...)`, including `border-radius token(animation.duration.fast) token(animation.curves.bezier.one)`.
- **D-08:** The final examples keep component CSS on explicit `token(...)` syntax.
- **D-09:** The final proof confirms custom `@keyframes` operate over the transition-safe property set.
- **D-10:** The final proof confirms the supported keyframe property set includes colors, opacity, dimensions, padding, margin, transform, font sizing, line height, gap, and insets.
- **D-11:** The final docs and parser tests cover percentage keyframe stops with intermediate percentages.
- **D-12:** The final docs and parser tests reject `from` and `to` aliases.
- **D-13:** The final docs state that keyframe blocks define property values while animation properties may reference `token(animation.*)`.
- **D-14:** The final docs state that keyframe stop values do not support theme token references in the first release.
- **D-15:** The final shell tests prove animations continue across rebuilds when stable `_mesh_key` and animation name remain present.
- **D-16:** The final shell tests prove a rebuild alone does not restart a still-present animation.
- **D-17:** The final renderer tests cover `animation-fill-mode`.
- **D-18:** The final renderer tests cover finite numeric iteration counts and `infinite`.
- **D-19:** The final renderer tests cover `animation-play-state: paused`.
- **D-20:** The final shell tests prove animation scheduling remains compatible with Phase 9 stable runtime-state rebuild behavior.
- **D-21:** The final parser tests reject keyframes containing unsupported properties.
- **D-22:** The final parser tests reject keyframes with no supported animatable properties.
- **D-23:** The final proof includes both compile/parse diagnostics and runtime diagnostics.
- **D-24:** The final parser tests cover static keyframe syntax and unsupported-property diagnostics.
- **D-25:** The final shell tests cover unresolved animation names and runtime token-resolution failures.
- **D-26:** The final style/runtime tests prove invalid animation token references fail hard with diagnostics.

<threat_model>
T-12-05: The implementation could be correct but undocumented or documented with stale `motion.*` and `from`/`to` examples, leading plugin authors to use unsupported syntax.
Mitigation: update docs and examples in the same phase, add grep-verifiable stale-term checks, and run all parser/style/render/shell animation tests before marking the phase ready.
</threat_model>
