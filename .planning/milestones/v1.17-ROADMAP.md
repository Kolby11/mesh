# Roadmap: MESH v1.17 Performance: Scripting VM Consolidation

**Archived:** 2026-06-07
**Status:** Shipped ✅
**Audit:** `.planning/v1.17-MILESTONE-AUDIT.md`

## Milestone Goal

Eliminate the per-component `mlua::Lua` VM allocation — the largest per-component startup and memory cost — by introducing a per-thread VM pool with `_ENV`-based environment isolation so each component gets a private namespace without paying the full stdlib + metatable cost. Lazy-init for inactive components; shared compiled chunks across runtimes once VM ownership is stable.

## Shipped Phases

### Phase 92: VM Pool Foundation
**Requirements**: POOL-01, POOL-02, POOL-03, POOL-04, CACHE-01, CACHE-02
**Plans**: 2 (pool.rs + chunk_cache.rs standalone types)
**Summary**: Thread-local `LuaVmPool`, `PooledVm` RAII guard, and process-wide `ChunkCache` keyed on FNV64 content hash. No behavioral change to existing `ScriptContext`.

### Phase 93: Host API Re-targeting
**Requirements**: ISO-02 (foundation), ISO-04
**Plans**: 2 (frontend + backend `install_host_api(&Table)` refactor)
**Summary**: `install_host_api()` accepts `&Table` target instead of hardcoding `lua.globals()`. `LuaVmPool` captures `pool_baseline_globals` snapshot at construction for later reactive state filtering.

### Phase 94: _ENV Isolation + Lazy-Init
**Requirements**: ISO-01, ISO-02 (completion), ISO-03, INIT-01, INIT-02, CACHE-03
**Plans**: 2 (ScriptContext restructure + checkin cleanup + cache wiring)
**Summary**: Replaced `ScriptContext::lua: Lua` with `vm: Option<PooledVm>` + `env_table: Option<Table>`. Added `ensure_initialized()` with lazy pool checkout, per-component `_ENV` table with `__index=globals()` metatable, host API installation into `_ENV`. Script chunks use `set_environment(env_table)`. Closures capture `env_table` for reactive state access. `uninit()` drops env and returns PooledVm to pool. `compile_and_execute` integrates `ChunkCache`.

### Phase 95: Integration + Validation
**Requirements**: INIT-03, INT-01, INT-02
**Plans**: 3 (BackendScriptContext lazy-init + frontend wiring + build verification)
**Summary**: `BackendScriptContext` defers `Lua::new()` via `ensure_lua()` gate. `FrontendSurfaceComponent::create_runtime_for_component` uses `ScriptContext::new_lazy()` + `compile_and_execute`. `reload_source()` evicts `ChunkCache` entries before `.mesh` recompile. Full workspace build passes with zero errors. 108 scripting tests pass, 25 backend tests pass.
