<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| T-39-01-01: contradictory relationship metadata is accepted and later tools cannot know whether an interface is a base, extension, or independent contract | high | Reject explicit `relationship`/`extends` contradictions in `MeshInterfaceDeclaration::validate`. |
| T-39-01-02: independent interfaces are blocked whenever a base exists for the same domain | high | Preserve existing `InterfaceGuidanceRecord` behavior as advisory only and add tests that independent same-domain declarations load. |
| T-39-01-03: provider declarations are mistaken for interface declarations | medium | Keep `InterfaceDeclarationNode` creation limited to `ModuleKind::Interface` plus `mesh.interface`; provider indexing remains separate. |
</threat_model>

<task type="auto">
  <name>Task 1: Harden explicit interface relationship validation</name>
  <files>crates/core/extension/module/src/package/module_manifest.rs, crates/core/extension/module/src/package/tests.rs</files>
  <read_first>crates/core/extension/module/src/package/module_manifest.rs, .planning/phases/39-contribution-and-interface-extension-index/39-RESEARCH.md</read_first>
  <action>Update `MeshInterfaceDeclaration::validate()` so explicit `relationship = InterfaceRelationship::Extension` requires a non-empty `extends`, explicit `relationship = InterfaceRelationship::Base` rejects `extends`, and explicit `relationship = InterfaceRelationship::Independent` rejects `extends`. Keep inferred behavior unchanged when `relationship` is omitted: `extends` infers extension, `mesh.*` infers base, otherwise independent. Add package tests for all three contradiction cases and one inferred extension case.</action>
  <verify>cargo test -p mesh-core-module package::tests interface_relationship</verify>
  <acceptance_criteria>
    - `relationship = "extension"` without `extends` produces a validation error mentioning `mesh.interface.extends`.
    - `relationship = "base"` with `extends` produces a validation error mentioning `mesh.interface.relationship`.
    - `relationship = "independent"` with `extends` produces a validation error mentioning `mesh.interface.relationship`.
    - Omitted `relationship` plus `extends` still produces `InterfaceRelationship::Extension`.
    - `cargo test -p mesh-core-module package::tests interface_relationship` exits 0.
  </acceptance_criteria>
  <done>Explicit interface relationship contradictions are blocking author errors.</done>
</task>
