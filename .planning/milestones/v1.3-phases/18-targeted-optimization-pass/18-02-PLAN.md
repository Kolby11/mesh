# Plan 18-02: Targeted Hotspot Optimization

<objective>
Optimize the one hotspot selected in `18-BASELINE.md` while preserving profiling-off behavior, visual output, benchmark contracts, and backend/service semantics.
</objective>

<requirements>
- `OPT-01`: At least one high-impact hotspot identified by the new profiler is optimized within `v1.3`.
- `OPT-03`: Normal shell behavior does not regress when profiling mode is off.
</requirements>

<decision_coverage>
- `D-01` and `D-02`: all optimization work reads and follows the baseline selection.
- `D-09` through `D-12`: all public behavior and contracts stay stable.
</decision_coverage>

<threat_model>
| Threat | Severity | Mitigation |
|--------|----------|------------|
| Optimization changes behavior rather than cost | high | Add regression tests for the selected path and preserve existing benchmark/profiling tests. |
| Benchmark contract is altered to fake improvement | high | Run `benchmark` tests and do not change scenario ids, labels, request names, or JSON shape. |
| Profiling-off path gains overhead or output | high | Run `profiling_` tests and add/adjust a profiling-off regression if the touched code path needs it. |
| Backend optimization adds service-specific Rust logic | high | Keep backend changes generic by interface/provider/stage only; do not parse audio payload fields. |
</threat_model>
