# Plan 14-01 - Debug Profiling Contract and Control Path

## Objective

Define the shared profiling snapshot contract and wire an explicit profiling toggle through the existing debug request path so Phase 14 stays debug-only and later phases inherit one stable control surface.

## Design Truths

- Profiling control must be shell-owned, typed, and debug-path only.
- Enabling profiling must reset the active session instead of appending to stale samples.
- Overlay visibility and profiling collection state must remain independent.

<threat_model>
T-14-01: Profiling can accidentally become a second diagnostics subsystem or a normal-user feature path if its contract and toggles live outside the current debug request flow.
Mitigation: extend `mesh-core-debug`, add explicit profiling request variants in `CoreRequest`, and route IPC/CLI controls through the same shell debug path that already owns overlay toggles.
</threat_model>

<steps>
