# Plan 16-04 - Benchmark Scaffold View, Final Inspector Proof, and Phase Verification

## Objective

Finish Phase 16 by adding the scaffold-only benchmark view, proving the full inspector on real surfaces, and writing the verification report tied back to the requirement set.

## Design Truths

- The benchmark view is informational in Phase 16, not executable.
- Final proof should cover all four inspector views plus the shell-level debug-path rules.
- The verification report must cite concrete tests, not manual confidence alone.

<threat_model>
T-16-04: The inspector can appear complete while either skipping the required benchmark view or pulling Phase 17 launcher behavior forward, and the phase can still close without explicit proof that empty states and debug-path rules survived the UI migration.
Mitigation: implement a scaffold-only benchmark view with exact category copy, add end-to-end shell/component regressions for all four views, and write a verification report that maps back to `PROF-01`, `INSP-01`, `INSP-02`, and `INSP-03`.
</threat_model>
