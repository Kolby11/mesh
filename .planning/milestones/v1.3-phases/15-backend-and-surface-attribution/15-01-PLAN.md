# Plan 15-01 - Backend Profiling Contract and Collector Extension

## Objective

Extend the shared debug profiling contract and bounded collector so backend provider/service attribution becomes a first-class snapshot payload while preserving stable per-surface timing summaries.

## Design Truths

- The backend view must be typed and explicit.
- Per-surface breakdowns must remain stable while backend data is added.
- All new attribution storage must stay inside the existing bounded collector.

<threat_model>
T-15-01: Backend timing can regress into one aggregate shell bucket or an ad hoc metadata blob, making hotspots non-actionable and forcing later phases to redesign the contract.
Mitigation: add first-class backend profiling snapshot/stage types in `mesh-core-debug`, extend the existing collector with bounded backend accumulators, and cover the contract with shell tests.
</threat_model>
