## Critical Gaps

### GAP-01: DMGE-03 — Debug overlay damage rect count (orphaned)

**Requirement:** Debug/profiling exposes damage rect count per frame alongside existing damage metrics.

No plan in Phase 101 claimed DMGE-03. Integration analysis shows `damage_rect_count` exists in `RetainedPaintSnapshot` (debug/src/lib.rs:230) and is serialized to the debug snapshot, but the value is computed as `u64::from(damage_area > 0)` — a binary 0/1 flag rather than the actual count of distinct `DamageRect` entries passed to the present path. The success criterion requires the count (e.g., "3 rects this frame"), not a boolean.

**Fix:** In the paint snapshot capture path, record `effective_damage.rects.len()` (the actual Vec length from `take_present_damage`) as `damage_rect_count` instead of the binary area-presence check. No new Wayland protocol work needed — the value is already available in the render dispatch.

### GAP-02: CR-01 — Blur region not cleared from compositor on backdrop-filter removal

**Requirement:** BLUR-04 — Blur region commits skipped cleanly when no backdrop-filter nodes exist.

**File:** `crates/core/presentation/src/wayland_surface/backend.rs:876-888`

When a surface previously had `backdrop-filter: blur(...)` and the node is removed, `compute_blur_region` returns `None`, `update_blur_region` stores `entry.blur_region = None`. On the next `present_with_damage` call, the `if let Some(region_rect) = entry.blur_region` guard skips the entire kde_blur block. No `kde_blur.set_region(None); kde_blur.commit()` is emitted. The compositor continues blurring the region indefinitely.

**Fix (from 103-REVIEW.md CR-01):**
```rust
// Add to SurfaceEntry:
pub(super) blur_committed: bool,

// In present_with_damage:
if let Some(ref kde_blur) = entry.kde_blur {
    match entry.blur_region {
        Some(region_rect) => {
            // ... existing set_region(Some(...)) + commit() ...
            entry.blur_committed = true;
        }
        None if entry.blur_committed => {
            kde_blur.set_region(None);
            kde_blur.commit();
            entry.blur_committed = false;
        }
        None => {}
    }
}
```
