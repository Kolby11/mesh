# 07 — Localization

> Part of the [MESH Specification](README.md).

Localization is **module-authored and user-extensible**: every module ships
translations for its own strings; **language packs** layer additional or
replacement catalogs on top (ordered multi-active chain, like icon packs);
the user picks a locale and fallback chain. Lookups go through the
`mesh.locale` interface — the same contract-and-registry model as everything
else.

**Status: shipped** (catalogs, `t()`, locale switching, fallback,
manifest-localized text); language-pack layering and some formatting surface
are **target** where marked.

## 1. Scoping

Translations are **scoped by module id**: `results.files` in
`@mesh/launcher` never collides with `results.files` in another module. This
is what makes third-party packs safe — they target modules individually.
Interface modules may also own catalogs (shared domain strings).

## 2. Module-bundled catalogs

```json
"mesh": {
  "i18n": { "defaultLocale": "en", "supportedLocales": ["en", "sk"] },
  "provides": {
    "i18n": [
      { "id": "en", "locale": "en", "path": "config/i18n/en.json" },
      { "id": "sk", "locale": "sk", "path": "config/i18n/sk.json" }
    ]
  }
}
```

- `mesh.i18n.supportedLocales` is install metadata (which language assets an
  installer may fetch/enable); `defaultLocale` is the module's terminal
  fallback and belongs in `supportedLocales`.
- `mesh.provides.i18n` lists the concrete bundled files. Any module kind may
  bundle catalogs for its own strings.

## 3. Language packs

A `language-pack` module contributes catalogs *for other modules* using the
same contribution shape, one entry per target module:

```json
{
  "name": "@community/cs-language-pack",
  "version": "1.0.0",
  "mesh": {
    "apiVersion": "0.1",
    "kind": "language-pack",
    "provides": {
      "i18n": [
        { "id": "launcher-cs", "locale": "cs", "module": "@mesh/launcher",
          "path": "translations/@mesh/launcher/cs.json" },
        { "id": "panel-cs", "locale": "cs", "module": "@mesh/panel",
          "path": "translations/@mesh/panel/cs.json" }
      ]
    }
  }
}
```

*(Status: target — the `module` targeting field and pack layering.)* Multiple
packs coexist as an ordered chain; catalogs from packs shadow module-bundled
catalogs key-by-key. Community translators can localize modules they don't
own; when a module renames keys, packs targeting it need updates.

## 4. Catalog format

Flat JSON, key → string; dotted keys are organizational, lookup treats them
as opaque.

```json
{
  "search.placeholder": "Hľadať aplikácie",
  "battery.charging": "Nabíja sa, {time_to_full}",
  "notifications.count": {
    "_plural": true,
    "one": "{count} upozornenie", "few": "{count} upozornenia",
    "many": "{count} upozornení", "other": "{count} upozornení"
  }
}
```

- **Interpolation**: `{name}` substitutes `args.name`; no expression
  evaluation in strings.
- **Plurals**: `_plural: true` objects with CLDR categories; `other`
  required.

## 5. Lookup chain

For `(module_id, key)`:

1. User-pinned override (settings store; rare, for corrections)
2. Language packs providing `(module_id, active_locale)`, chain order
3. Module-bundled catalog for the active locale
4. Repeat 2–3 for each locale in the user's fallback chain
5. Module-bundled `defaultLocale` catalog
6. Visible miss: the key rendered as `!!key` plus a diagnostic

Locale codes are BCP 47; matching is most-specific-first (`sk-SK` satisfies
`sk`; not vice versa unless the chain says so). User settings
([08](08-settings.md)):

```json
"shell": { "locale": { "active": "sk-SK", "chain": ["sk-SK", "sk", "en"] } }
```

When `chain` is omitted it derives from `active` (drop region → drop script →
`en`). Changing locale emits `LocaleChanged`; every subscriber refreshes.

## 6. Consumption surfaces

**Scripts** — the i18n library and locale interface:

```luau
local t = import("mesh.i18n", "t")
local loc = require("mesh.locale")

label = t("search.placeholder")                 -- module-scoped key
loc.format_number(1234567.89)                    -- "1 234 567,89" in sk-SK
loc.format_date(os.time(), "short")
loc.format_duration(3675)                        -- "1 h 1 min"
```

Modules never hand-format numbers/dates/durations — CLDR rules differ per
locale. *(Formatting surface beyond `t`: target.)*

**Manifests** — field-local localized text objects wherever a manifest field
should be catalog-backed (keybind labels, prop labels):

```json
"label": { "t": "keybind.mute.label", "fallback": "Mute audio" }
```

Raw strings in manifest text fields are literals, never implicit keys.
`fallback` is required and used when no catalog resolves; misses are
non-fatal diagnostics with module id, field path, key, and fallback.

**Props** — `label`/`description`/enum labels in `<props>` are
`LocalizedText` resolved the same way ([03 §3](03-components.md)). The
installed graph scans `.mesh` sources for `t()` keys and reports missing
catalog entries.

## 7. RTL, bidi, and script fallback

- `locale.dir` exposes `"ltr"`/`"rtl"`; surfaces flip automatically for RTL
  locales. Authors use logical properties (`margin-inline-start`), not
  physical ones.
- Font fallback for scripts the active fonts lack is the renderer's job
  (cosmic-text); themes may declare script-specific stacks later
  ([06 §8](06-fonts.md)).

*(Status: target.)*

## 8. Tooling

```
mesh locale list                    # locales available (bundled + packs)
mesh locale active                  # current locale + chain
mesh locale set <code>
mesh locale which <module> <key>    # which layer supplied the string
mesh locale missing <module>        # keys with no catalog entry
mesh locale extract <module>        # translator template with every used key
```
